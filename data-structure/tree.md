# 树与二叉树

> **考情提示**：树与二叉树是数据结构中**分值最高的单章**，408 每年至少 2 道选择题（4 分）+ 几乎必出一道大题（≈10 分）。核心考点：二叉树遍历（递归/非递归）、遍历序列构造二叉树、线索二叉树、哈夫曼树 WPL 计算。本章是第一轮复习的重中之重。

## 树的基本概念

树是 n（n ≥ 0）个结点的有限集合——空树 n = 0，非空树有且只有一个**根**结点，其余结点可以划分为 m 个互不相交的有限集合，每个集合本身又是一棵树，称为根的**子树**。注意这里的关键词是「互不相交」——树中不能有环路，每个结点只能属于一棵子树。

几个容易混淆的术语：结点的**度**是它拥有子树的个数，树的**度**是树中所有结点度的最大值。**叶子结点（终端结点）** 的度为 0。树中结点**层次**从根开始算——根是第 1 层（注意：有的教材从 0 开始，408 从 1 开始）。**深度**是结点的最大层次。**有序树**指子树有左右顺序之分，不能互换；**无序树**则反之。

树的性质是选择题高频素材，务必记住：
- 树中的结点数 = 所有结点的度数之和 + 1（每条边贡献一个度）
- 度为 m 的树中第 i 层最多有 m 的 (i-1) 次方个结点
- 高度为 h 的 m 叉树最多有 (m^h - 1)/(m - 1) 个结点
- 具有 n 个结点的 m 叉树的最小高度为 ⌈log_m(n(m-1)+1)⌉

> 💀 考研基本不考「树的广义表表示法」和「双亲表示法的具体代码实现细节」。

## 二叉树

### 二叉树的定义和主要特征

二叉树 🔥 是每个结点至多有两棵子树（左子树和右子树），且子树**有严格左右之分**的树形结构。二叉树 ≠ 度为 2 的树——度为 2 的树要求至少 3 个结点（根 + 两个孩子），而二叉树可以只有左孩子、只有右孩子、甚至空树。二者是两个完全不同的概念（考卷上常见混淆选项）。

二叉树的核心性质（必背）：
1. 第 i 层最多 2^(i-1) 个结点（i ≥ 1）
2. 高度为 h 的二叉树最多 2^h - 1 个结点
3. 叶子结点数 n₀ = 度为 2 的结点数 n₂ + 1（来自等式 n₀ + n₁ + n₂ = 2n₂ + n₁ + 1）
4. 具有 n 个结点的完全二叉树的深度为 ⌊log₂n⌋ + 1
5. 对完全二叉树按层编号（根为 1），编号为 i 的结点——其左孩子为 2i，右孩子为 2i+1，父结点为 ⌊i/2⌋

几种特殊二叉树的区分：
- **满二叉树**：每一层都被完全填满，深度为 h 时有 2^h - 1 个结点。它是完全二叉树的子集。
- **完全二叉树**：除最后一层外每层都填满，最后一层所有结点靠左排列。性质 5 就是完全二叉树的特性，也是堆排序的基础。
- **二叉排序树（BST）**、**平衡二叉树（AVL）** 属于「树的应用」类，后面查找章节详讲。

### 二叉树的存储结构

**顺序存储**（数组）🔥 只为完全二叉树而生。对完全二叉树按层编号，结点 i 存在数组下标 i 处（或 i-1），通过性质 5 直接算父子关系，无需额外指针。但对于一般的非完全二叉树，顺序存储会大量浪费空间——数组中需要给不存在的结点留空位。

**链式存储（二叉链表）**🔥 是考研和实际中最通用的方式。每个结点有三个域：`data`、`lchild`（左孩子指针）、`rchild`（右孩子指针）。有 n 个结点的二叉链表共有 n+1 个空指针域（因为 n 个结点有 2n 个指针，n-1 条边，空指针 = 2n - (n-1) = n+1）。这 n+1 个空指针正是「线索二叉树」优化的切入口。

> 🔥 实际开发中二叉树几乎都是链式存储（C++ 的指针、Java 的引用、Python 的对象属性），顺序存储的二叉树只在堆排序和完全二叉堆中才用。

## 二叉树的遍历

### 先序遍历（NLR）

先序遍历的规则在考研中表述固定：如果二叉树非空，**先访问根结点，再先序遍历左子树，再先序遍历右子树**——「NLR」。三个字：根-左-右。对于一个给定二叉树，要能画出首次访问每个结点的顺序路线。

递归写法极为简洁，非递归写法需要手动维护一个栈：先把根入栈，然后循环——弹出一个结点并访问，如果它有右孩子先让右孩子入栈，如果它有左孩子再让左孩子入栈（注意入栈顺序和访问顺序是反的，因为栈是后进先出）。

> 在考研大题中，先序遍历的非递归写法出现频率显著高于中序和后序，因为它不需要额外的「已访问」跟踪。

### 中序遍历（LNR）

中序遍历的规则是「LNR」：先中序遍历左子树，然后访问根，然后中序遍历右子树。**对于一棵二叉排序树（BST），中序遍历的结果一定是递增有序序列**，这是中序遍历最重要的性质，也是查找章节 BST 排序的基础。

非递归中序遍历是所有三种遍历中最需要多练的一个。核心思路：一直沿着左孩子方向走，沿途全部压栈，走到不能再左后——弹出一个结点访问它，然后把当前指针指向它的右孩子，重复以上过程。这段代码一定要背，408 真题考过手写非递归中序。

### 后序遍历（LRN）

后序遍历是「LRN」：先后序遍历左子树，再后序遍历右子树，然后访问根。后序的非递归实现是所有三种中最麻烦的，因为需要额外判断「刚才是从左子树回来的还是从右子树回来的」——通常在栈里面同时保一个标记变量，或者在结点结构中增设 visited 计数。

后序遍历在考研中有一个重要应用：计算树的高度、释放树的动态空间。因为必须先释放完左右子树才能释放根结点。代码题里出现「自底向上的操作」时，往往就是后序。

### 层次遍历

层次遍历 🔥 不通过递归，而是用一个队列辅助。根入队 → 反复出队并访问 → 左右孩子入队（如果存在） → 直到队列为空。这个过程和图的 BFS 一模一样。层次遍历在考研里可以考：
- 直接写层次遍历的算法（送分题级别，代码简单）
- 基于层次遍历完成某个功能：统计树的宽度（找出哪层结点最多）、求树的高度、判断是否完全二叉树

### 由遍历序列构造二叉树

**先序 + 中序** → 唯一确定二叉树 🔥：先序序列的第一个元素就是整棵树的根，在中序序列中找到这个根——中序里根左边的元素全是左子树，右边全是右子树。然后对左右子树递归重复以上过程，直到序列为空。

**后序 + 中序** → 唯一确定二叉树 🔥：后序序列的**最后一个**元素是根，在中序中找到根的位置，左边是左子树，右边是右子树。然后在后序中把对应的段截出来继续递归。

**层序 + 中序** → 唯一确定二叉树 ⚠️：也成立，但不如前两种高频。层序的第一个元素是根，根在中序左边的元素按层序出现的顺序就是左子树的构建序列。

> **考研必考结论**：只知道先序和后序（没有中序），不能唯一确定二叉树！考试中经常作为选择题陷阱出现。

> 💀 考研基本不考「只给层序和先序/后序能不能唯一确定二叉树」这类特殊变体。

## 线索二叉树

### 线索二叉树的基本概念

二叉链表有 n+1 个空指针域，线索二叉树 🔥 就是把这些空余的指针利用起来——让左孩子为 NULL 的指针指向该结点的**前驱**（某种遍历方式下之前的结点），让右孩子为 NULL 的指针指向**后继**。这时原本空的指针就不再是 NULL，而是「线索」。为了区分「这是真正指向孩子的指针还是线索」，结点还需要增设两个标志位：`ltag` 和 `rtag`，值为 0 表示指向孩子，值为 1 表示指向线索。

> 🔥 线索二叉树在数据库系统（如 B+ 树的叶子链表）和文件系统中都有应用，其核心思想「用指针替代递归/栈来找下一个」在考研中也是理解难点。

### 线索二叉树的构造

以**中序线索化**为例（最常考）：递归遍历整棵树，设置一个全局 `pre` 指针始终指向前一个访问过的结点。每访问一个新结点时，如果该结点的左孩子为空，设 `lchild = pre` 且 `ltag = 1`；如果 pre 的右孩子为空，设 `pre->rchild = 当前结点` 且 `pre->rtag = 1`；然后 `pre = 当前结点` 继续遍历。

**先序线索化** 和**后序线索化**的原则是一模一样的——核心区别只在于先/中/后序遍历的访问顺序不同，线索化时「pre」指针的跟进时机也随之变化。

> ⚠️ 考研考得最多的是中序线索二叉树，先序和后序线索化了解基本流程即可。

### 线索二叉树的遍历

线索二叉树的优势在这里才体现出来：**不用栈也不用递归**，可以直接从一个结点找到它的后继。以中序线索二叉树为例：如果一个结点的 `rtag == 1`，后继直接就是 `rchild`；如果 `rtag == 0`，则后继是其右子树上最左下的那个结点（沿右孩子的左链一路往下走，直到 `ltag == 1`）。前驱同理。

遍历中序线索二叉树（顺序输出所有结点）的代码极为简洁：从根出发先走到最左边结点，然后反复找当前结点的后继输出，直到后继为空。整个过程 O(n)，空间复杂度 O(1)，这是普通二叉树无法做到的。

> 💀 考研基本不考「中序线索二叉树中添加/删除一个结点」的具体代码操作。知道遍历怎么走、前驱后继怎么找即可。

## 树、森林

### 树的存储结构

树的存储主要有三种方式：

- **双亲表示法** 🔥：每个结点只记录它的父结点下标（一个 `parent` 字段），整棵树存在一个结点数组中。优点是找父结点 O(1)，缺点是找孩子必须遍历整个数组 O(n)。这在并查集中就是标准存储方式。
- **孩子表示法**：每个结点维护一个孩子链表，把所有孩子的下标链起来。找孩子 O(孩子数)，找父结点则困难。💀 这个表示法现在基本只在教材中出现，实际开发很少直接使用。
- **孩子兄弟表示法** 🔥（左孩子右兄弟）：每个结点只留两个指针——一个指向第一个孩子，一个指向下一个兄弟。这种方法能把任何树唯一地转换成二叉树。它就是树和二叉树之间的「通用转换器」，复杂度 O(1)。

### 树、森林与二叉树的转换

**树转二叉树** 🔥 的规则叫「左孩子右兄弟」：对树中任意一个结点，它的第一个孩子变为这个结点在二叉树中的左孩子，每一个兄弟变为二叉树中的右孩子。森林转二叉树则是先把每棵树各自转成二叉树，再将后一棵二叉树的根作为前一棵二叉树根的右孩子接上去。这个规则是 408 大题中画图题的火热素材——给你一棵树（或一个森林），让你画出对应的二叉树。

反过来，**二叉树还原为树**的规则是：二叉树中某结点的左孩子是它在原树中最左边的孩子，右孩子是它在原树中的下一个兄弟。还原为森林的方法是先把二叉树根沿右链断开——每个根的整棵左子树是一棵树——然后分别对每棵子树再还原成树。

> 💀 考研基本不考「二叉树还原为森林」的复杂边界情况。掌握树转二叉树和森林转二叉树的正向转换就够了，大题都是正向考查的。

### 树和森林的遍历

树有两种遍历方式：**先根遍历**（先访问根，再依次先根遍历每一棵子树）和**后根遍历**（先依次后根遍历每一棵子树，再访问根）。一条非常重要的结论——这在选择题里百试百灵——**树的先根遍历序列等于它对应二叉树的先序遍历序列，树的后根遍历序列等于它对应二叉树的中序遍历序列**。

森林的遍历也是类似的：先序遍历森林挨个先序遍历每棵树，中序遍历森林挨个后根遍历每棵树（森林的中序遍历等于对应二叉树的先序还是什么？答案：森林的中序遍历等于对应二叉树的中序遍历）。这个转换关系在考试中只要能背，不需要自己重新推导。

> 🔥 实际中，文件系统的目录结构就是一棵树，操作系统的文件浏览器执行的就是先根遍历。

## 树与二叉树的应用

### 哈夫曼树和哈夫曼编码

哈夫曼树（最优二叉树）🔥 是 408 每年必考选择题的固定项目。给定 n 个权值，构造一棵二叉树使得带权路径长度（WPL）最小——带权路径长度等于每个叶子结点的权值乘以它到根的路径长度之和。构造算法很简单：将所有结点放入一个小根堆，每次取出权值最小的两个结点合并成一个新内部结点（权值为两者之和），不断重复，直到堆中只剩一个结点。合并过程中形成的二叉树就是哈夫曼树——带权路径长度为最小。

哈夫曼树没有度为 1 的结点——这是它的一条关键性质：由 n 个叶子结点构成的哈夫曼树共有 2n-1 个结点。选择题经常会在这里做文章：给你 n 个叶子，问哈夫曼树的结点总数、或者你需要申请多少个存储单元。

**哈夫曼编码** 🔥 是哈夫曼树最经典的应用：给使用频率不同的字符分配不等长二进制编码——频率越高分配越短的编码。所有叶子都从根向左走代表 0、向右走代表 1，从根一路走到某个叶子的二进制序列就是该字符的编码。哈夫曼编码是**前缀编码**——任意一个字符的编码都不是其他字符编码的前缀——所以解码时不会产生歧义。

> 🔥 哈夫曼编码至今仍被广泛应用：JPEG 图像压缩、MP3 音频压缩、DEFLATE（ZIP/gzip/PNG）的背后都有哈夫曼编码的身影。它不是一个过时算法，是考研中少数既考理论又有极高实用价值的知识点。

> **408 必考**：给一组字符和频率，要求你画出哈夫曼树、写出每个字符的编码、计算 WPL 值。选择题中 WPL 是具体数字、判断题中检验一张编码表是或否是哈夫曼编码。

### 并查集

并查集（Disjoint Set / Union-Find）🔥 是一种专门处理不相交集合的合并和查询操作的数据结构，图论中的最小生成树 Kruskal 算法就依赖它。它存储在一个整型数组中：`parent[i]` 表示元素 i 的父结点下标，如果 `parent[i] == -1`（或等于自身），则 i 是集合的根。

并查集的核心操作只有两个。**Find(i)**：查找元素 i 所在的集合（谁是大王），一路顺着 parent 链往上直到根。**Union(i, j)**：合并元素 i 和 j 所在的两个集合，找到两个根后把其中一个根的 parent 指向另一个。如果每步 Find 都顺着链爬上树顶，最坏情况会退化成 O(n)。

**路径压缩** 🔥 是并查集最重要的优化——在执行 Find(i) 的过程中，把返程路径上每一个点直接挂到根下面。这样后续的查找可以直接命中，近似 O(1)。加上 Union 时的「按秩合并」优化，整体的均摊复杂度接近**反阿克曼函数**，在实践中完全等同于常数时间。考研中一般只要求理解路径压缩的基本思想并写简单版本的 Find 代码。

> 🔥 并查集的代码非常短（Find 函数一般不到 5 行），是 408 大题中最容易拿分的代码题之一。Kruskal 算法的实现中并查集是标准件。

## 常见题型

- **二叉树遍历的递归/非递归算法**：非递归中序是重点，建议背到可以闭眼写出。非递归后序的「两次入栈法」或「标记法」掌握一种即可。
- **遍历序列反构二叉树**：给先序+中序或后序+中序，动手在纸上画出树的整个过程，408 每年至少考一次。
- **线索二叉树的遍历**：给一个中序线索二叉树的图，让你指出某个结点的前驱或后继是谁。
- **哈夫曼树的 WPL 计算**（高频）：两种考法——一种是手动构造哈夫曼树再算 WPL，一种是直接套公式用带权叶子路径累加计算，注意加权再求和不要漏项。
- **完全二叉树的性质** 🔥：用编号公式算父结点、左孩子、右孩子位置，以及计算叶子结点数（n₀ = n₂ + 1，结合 n = 2n₀ - 1 等变形），是选择题的送分 / 送死临界区。
