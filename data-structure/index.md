# 数据结构概述

> **考情提示**：本章在 408 试卷中一般以 1–2 道选择题出现，分值 2–4 分。重点考查时间复杂度计算和逻辑结构/存储结构的区分，属于「送分题」，务必拿满。

## 数据结构的基本概念

### 基本概念和术语

**数据（Data）** 是对客观事物的符号表示，在计算机中是指所有能输入并被处理的符号的总称。**数据元素（Data Element）** 是数据的基本单位，通常作为一个整体处理——比如学生表里的一行就是一个数据元素。**数据项（Data Item）** 是构成数据元素的不可分割的最小单位，例如一个学生的学号、姓名各自是一个数据项。**数据对象（Data Object）** 是具有相同性质的数据元素的集合，是数据的一个子集。

**数据类型（Data Type）** 是一个值的集合和定义在这个值集上的一组操作的总称。C 语言的 `int` 就是这样一个类型：取值范围有限的整数 + 加减乘除取模等操作。**抽象数据类型（ADT）** 🔥则在数据类型的基础上进一步抽象，只关心它的逻辑特性（能做什么），不关心实现细节（怎么做）。考研 408 中 ADT 的概念常在选择题选项中出现，能用一句话描述一个 ADT 对应对你来说就够了：ADT = 数据对象 + 数据关系 + 基本操作。

### 数据结构三要素

这门课之所以叫「数据结构」，核心就在于以下三个要素：

- **逻辑结构**：数据元素之间的逻辑关系，与计算机无关。它分为：
  - **线性结构**：元素之间一对一，如线性表、栈、队列、数组
  - **非线性结构**：集合（无关系）、树形（一对多）、图形（多对多）
- **存储结构（物理结构）**：逻辑结构在计算机内存中的实际存放形式。主要有四种：
  - **顺序存储** 🔥：逻辑相邻的元素物理上也相邻（如数组），随机存取效率极高
  - **链式存储** 🔥：元素物理上随意，通过指针链接，插入删除高效
  - **索引存储**：建立索引表指向元素位置
  - **散列存储**：根据关键字直接通过散列函数计算地址
- **数据的运算**：施加在数据上的操作定义，包括运算的定义（逻辑层面，面向用户）和运算的实现（物理层面，面向计算机）

> **考研出题套路**：常考「以下哪种说法的逻辑结构不同/相同」，关键是把逻辑结构和存储结构分开想——你先想元素之间的关系（一对一？一对多？），然后再想怎么存。

## 算法和算法评价

### 算法的基本概念

算法的定义很简单：**对特定问题求解步骤的一种描述**，表现为指令的有限序列。一个合格的算法必须具备五个特性：**有穷性**（有限步后必须结束，不能死循环）、**确定性**（每一步有确切定义，不能有二义性）、**可行性**（每一步都能通过有限次基本运算完成）、**输入**（可以有零个或多个）、**输出**（必须有一个或多个）。

好算法的目标则是：**正确性**（能正确处理各种输入）、**可读性**（方便人理解维护，不是写天书）、**健壮性**（能对非法输入做出反应而不会崩溃）、**效率**（时间复杂度和空间复杂度低）。考研中，**正确性是底线**，但选择题和代码题的核心考查对象是效率——给定一个算法，你要能分析它的时间复杂度和空间复杂度。

### 算法效率的度量

**时间复杂度** 🔥 是考试出镜率最高的概念。它不计算算法实际运行了多少秒，而是关注**基本操作的执行次数随着问题规模 n 的增长而增长的趋势**。用的是渐进时间复杂度，记作 **O(f(n))**。比如一个算法的基本操作执行次数为 2n² + 3n + 1，我们就说它的时间复杂度是 O(n²)——只保留最高阶，系数全扔掉。

常见的时间复杂度从小到大排列为：

> O(1) < O(log₂n) < O(n) < O(n log₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)

**空间复杂度** 🔥 类似，衡量的是算法运行过程中额外占用的辅助存储空间。注意这里的关键词是「额外」——算法使用的输入数据本身占的空间不算在空间复杂度里，只算**附加空间**。比如冒泡排序的辅助空间只需一个临时变量，空间复杂度就是 O(1)；而归并排序需要大小为 n 的辅助数组，空间复杂度就是 O(n)。

> 💀 **考研基本不考**：原地工作（in-place）的严格定义、递归算法栈空间是否计入空间复杂度的学派争论。跟着王道/天勤的做法即可，把递归栈算入空间复杂度。

## 常见题型

- **时间复杂度计算**（高频）：给你一段循环/递归代码，让你分析 O 值。注意多层嵌套循环的外层和内层无关时直接乘，相关时累加。
- **逻辑结构与存储结构的区分**：选择题常考「队列的逻辑结构和存储结构分别是什么」这类概念题，牢记逻辑结构是抽象的（线性/非线性），存储结构是具体的（顺序/链式）。
- **数据三要素的应用**：选项里经常会混搭错误概念（如「存储结构就是逻辑结构」），能用三要素的框架去排除即可。
