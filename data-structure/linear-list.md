# 线性表

> **考情提示**：本章是数据结构最基础的一章，但也是**综合应用题的重要出题来源**。顺序表的插入/删除移动次数、链表逆置、链表合并等算法都是大题的热门考点。408 一般会在选择题中考核 1–2 题（2–4 分），偶尔直接考链表操作的代码题（≈10 分）。

## 线性表的定义和基本操作

线性表 🔥 是数据结构中最简单的线性结构。它具有相同数据类型的 n 个数据元素的有限序列——「有限」意味着元素个数可数，「序列」意味着元素间有先后次序。线性表是一个逻辑结构的概念，和它的物理实现方式（顺序存储还是链式存储）无关。

线性表的基本操作是整个数据结构的通用语言。无论后续学栈、队列、树还是图，你会发现许多操作的名字都一样，语义也相通。核心操作包括：**初始化**（InitList）、**销毁**（DestroyList）、**判空**（ListEmpty）、**求表长**（Length）、**按值查找**（LocateElem）、**按位查找**（GetElem）、**插入**（Insert）、**删除**（Delete）。此外还有输出、排序等辅助操作。考试中最重要的就是**插入、删除和查找这三个**——尤其是按位查找和按值查找的区别，常常在分析时间复杂度时考察。

## 顺序存储

### 顺序表的定义

用一组地址连续的存储单元依次存放线性表中的数据元素——这就是顺序表 🔥。C 语言里直接用一维数组实现，实际上就是一个封装了数组和当前长度的结构体。因为地址是连续的，所以顺序表最大的优点是**支持随机存取**——知道下标就直接通过 `a + i × sizeof(ElemType)` 访问，时间复杂度 O(1)。

> 💀 **考研基本不考**：动态分配数组（C++ 的 `vector`、Java 的 `ArrayList`）的内部扩容机制。考试中的顺序表默认是静态数组，最大容量预先固定。

### 顺序表基本操作的实现

**插入操作**：在位置 i —（假设从 1 开始编号）插入一个新元素，需要把第 i 到第 n 个元素统一往后挪一位，然后放入新元素，表长加一。时间复杂度为 O(n)，因为最坏情况（i = 1）要移动所有 n 个元素。考计算时记住结论：插入到位置 i 的概率为 1/(n+1) 时，平均移动次数 = **n/2**。

**删除操作**：删除位置 i 处的元素，把第 i+1 到第 n 个元素全部向前挪一位，表长减一。时间复杂度也是 O(n)，平均移动次数 = **(n-1)/2**。

**按值查找**：从头到尾扫描一遍比较，找到返回位置，找不到返回 0。这就是**顺序查找**，时间复杂度 O(n)。

> 这三点务必背熟，考研大题常见套路是「在有序顺序表中插入/删除元素」，此时虽然查找可以用二分优化到 O(log n)，但移动元素的开销依然是 O(n)，总时间复杂度还是 O(n)。

## 链式存储

### 单链表的定义

单链表 🔥 通过一组任意的存储单元存放线性表的数据元素——这组单元可以不连续，每个结点除了存放数据（data 域）还存放指向下一个结点的指针（next 域），形成一条链。**头结点**是单链表中很重要的设计惯例：它是一个不存放数据的附加结点，插在第一个有效结点之前。带头结点的好处是让插入和删除操作统一——不用特殊处理空表或者第一个位置。

考研中画图是基本功：每个结点画成一个小方框（分两格——data 和 next）→箭头指向下一个结点→最后一个结点的 next 指向 NULL。能画出插入/删除前后指针的变化，代码就不容易写错。

### 单链表基本操作的实现

**建立单链表**有两种方式：**头插法**和**尾插法**。头插法每次新结点插到头部，最终链表中的顺序和输入顺序相反——这是链表逆置的基础。尾插法则需要一个始终指向尾结点的指针，每次新结点接到尾部，保持输入顺序。

**按序号查找**：从第一个结点开始数，数到第 i 个返回指针，时间复杂度 O(n)。**按值查找**：从第一个结点开始逐个比较 data，复杂度也是 O(n)。注意这两种查找方式都无法发挥随机取存的优势，因为链表没办法直接跳到第 k 个元素——只能顺着链一个一个走。

**插入结点**是考试最高频的操作。在结点 p 之后插入新结点 s 只需两步：`s->next = p->next; p->next = s;`。在 p 之前插入则需要先找到 p 的前驱（复杂度 O(n)），或者用后插法：先把 s 插到 p 后面，然后交换 s 和 p 的 data 值（O(1)，但前提是 data 可交换）。

**删除结点**同理：删除 p 的后继只需 `q = p->next; p->next = q->next; free(q);`。删除 p 本身则必须找到前驱，或者把 p 的后继的 data 复制到 p 后再删除后继（同样是 O(1) 的小技巧，但考试里写法要小心）。

### 双链表

双链表在每个结点中增加了一个 **prior 指针**指向前驱，所以可以双向遍历。插入时需要注意修改四个指针：先修改新结点 s 的 prior 和 next → 再修改 p 的后继的 prior → 最后修改 p 的 next。顺序千万不能错，只要记住「先连新的，再断旧的」这个原则。

双链表的**删除**比单链表简单——因为可以直接找到前驱，不需要遍历。但依然需要修改两个指针：`p->prior->next = p->next; p->next->prior = p->prior;`。

> 🔥 双链表在很多实际系统中是标配（Linux 内核中的链表、Java 的 LinkedList、LRU Cache 用到的双向链表），考研中双链表的操作题不如单链表多，但概念题（如「双链表能否直接访问前驱」）偶尔出现。

### 循环链表

循环链表 🔥 与普通链表唯一的区别是**最后一个结点的 next 指针指向头结点**（而不是 NULL），形成一个环。这样从任意结点出发都能遍历整个链表。

**循环单链表**通常只设尾指针（不设头指针），因为通过尾指针可以 O(1) 访问尾部和头部（`tail->next` 就是头结点），适合需要频繁在两端操作的场景，比如队列的实现。

**循环双链表**则将头结点的 prior 也指向尾结点，形成双向环。判空条件是 `L->next == L` 或 `L->prior == L`。

### 静态链表

**静态链表** 💀 用数组来描述单链表——数组的每个元素有两个域：data 和 next（存放后继元素的下标，而不是指针），用 -1 表示结尾。它出现在早期的编程语言（如 Fortran）中，当时没有指针类型。**考研中几乎只在早年选择题中出现**：你需要知道静态链表的 next 域存的是**数组下标**而不是地址，判空条件是某个预定义的下标值为 -1。

> 💀 **考研基本不考**：静态链表的具体操作代码。知道概念（数组实现链表、next 域是下标）即可。

## 顺序表和链表的比较

| 对比维度 | 顺序表 | 单链表 |
|----------|--------|--------|
| 存取方式 | 随机存取 O(1) | 顺序存取 O(n) |
| 逻辑与物理关系 | 逻辑相邻 = 物理相邻 | 逻辑相邻 ≠ 物理相邻 |
| 插入/删除 | O(n)，主要花在移动元素 | O(1)（已知位置），O(n) 花在查找 |
| 空间分配 | 静态分配，可能浪费或溢出 | 动态分配，无溢出但每个结点多一个指针 |
| 存储密度 | 高（只有数据） | 略低（数据 + 指针） |
| 适用场景 | 频繁查找，很少增删 | 频繁增删，很少按位置查找 |

> **考研结论**：选择顺序表还是链表，取决于操作特征——「查多用顺序，改多用链表」。大题中如果题目说「元素个数未知且大量增删」，那就是暗示要用链表。

## 常见题型

- **顺序表插入/删除的移动次数计算**（高频选择题）：牢记平均移动 n/2（插入）和 (n-1)/2（删除），有序表插入/删除额外算上查找时间。
- **单链表插入/删除的指针操作序列**（高频选择题 + 代码题）：给你一句话描述插入操作，让你选哪个代码序列正确——画图，先连新指针再断旧指针。
- **链表逆置**（高频大题）：头插法是标准解法，每次摘下原链表第一个结点插入新链表头部。时间复杂度 O(n)，空间复杂度 O(1)。
- **双链表的插入和删除顺序**：四个指针修改步骤，错一步就全乱。先把新结点的前后指针指好，再去改周边结点的指针。
