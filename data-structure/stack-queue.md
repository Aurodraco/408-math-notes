# 栈和队列

> **考情提示**：栈和队列是限制性线性结构，408 选择题几乎每年必考（2–4 分），重点是出栈序列的合法性判断、循环队列判空/判满、表达式转换。大题一般不单独考栈队列本身，但它们是后续算法（树遍历、图搜索、递归转非递归）的基础工具，必须掌握。

## 栈

### 栈的基本概念

栈 🔥 是一种**只在一端进行插入和删除**的线性表，这一端称为**栈顶**，另一端称为**栈底**。把它想象成一摞盘子——你只能从最上面放或取。这种操作限制产生了一个核心特性：**后进先出（LIFO，Last In First Out）**。最后入栈的元素最先出栈，而栈底的元素只有等它上面的所有元素都出栈后才能出来。

栈的基本操作只有六个：初始化（InitStack）、判空（StackEmpty）、进栈（Push）、出栈（Pop）、读栈顶（GetTop）、销毁（DestroyStack）。注意**读栈顶不等于出栈**——读栈顶只是看一眼栈顶元素是什么（常出现在判断操作是否合法的选择题中），而出栈会把栈顶元素弹出来并返回。

💀 考研基本不考「栈和一般线性表哪个更强大」这类哲学题。答案永远是一般线性表——因为栈只是特殊线性表，能做线性表的子集操作。

### 栈的顺序存储

顺序栈 🔥 是考试和实际开发中最常用的实现方式。用一个数组 `data[MAXSIZE]` 和一个整型 `top` 指针来管理。`top` 指向栈顶元素的位置——初始化时 `top = -1` 表示空栈，进栈 `top++` 后放入元素，出栈取元素后 `top--`。判空条件是 `top == -1`，判满条件是 `top == MAXSIZE - 1`。

关于 top 指针的指向，王道和天勤教材有时写法不同——有的规定 `top` 指向栈顶元素的下一个空位（此时初始化 `top = 0`，进栈先放后移）。两种写法本质上没区别，只是组织习惯的差异。**考研中只要确保自己的代码自洽即可**，不用纠结哪种「对」。

**共享栈**是利用顺序栈的「栈底固定不动，栈顶动态变化」的特点，让两个栈共享一块连续空间。两个栈的栈底分别设在数组的两端，两个栈顶向中间增长。只有两个栈顶指针相遇（`top1 + 1 == top2`）才算栈满。这种设计在实际中很少用了 🔥⚠️，但考研选择题偶尔出现 —— 你需要能计算共享栈的最大容量，以及判断某个位置属于哪个栈。

### 栈的链式存储

链栈 🔥 就是一个没有头结点的单链表，栈顶对应链表头部。因为链表头部插入和删除都是 O(1)，天然适合栈。链栈不存在满的问题（除非内存耗尽），适合栈容量不可预测的场景。进栈就是在头部插入新结点，出栈就是摘除头部结点。

链栈在考研中出镜率远低于顺序栈。大多数考题默认是用数组实现的顺序栈，只有明确提到「用链表实现」时才考虑链栈。

> 🔥 实际开发中栈的实现基本由标准库提供（C++ 的 `std::stack`、Java 的 `Stack`/`Deque`、Python 的 `list`），不需要手写。但考研会考手写，这是测试你对指针操作和数组边界控制的基本功。

## 队列

### 队列的基本概念

队列 🔥 是一种**只允许在一端插入、另一端删除**的线性表。插入端叫**队尾（rear）**，删除端叫**队头（front）**。它和生活中排队一样——先来的先服务，这就是**先进先出（FIFO，First In First Out）**。队列有两个关键指针：`front` 指向队头元素，`rear` 指向队尾元素（或队尾的下一个位置，取决于实现约定）。

队列的基本操作也是六个：初始化（InitQueue）、判空（QueueEmpty）、入队（EnQueue）、出队（DeQueue）、读队头（GetHead）、销毁（DestroyQueue）。这些操作在后续章节会反复用到——二叉树的层次遍历用队列辅助、图的 BFS 用队列辅助、操作系统中进程的就绪队列也是队列。

### 队列的顺序存储

直接用普通数组实现顺序队列有个严重问题——**假溢出**。入队时 `rear++`，出队时 `front++`，一段时间后 `rear` 到了数组末尾，但 `front` 前面可能还有大量空位置没用过，此时数组「装满了」实际上只是 `rear` 走到了尽头。解决假溢出的唯一方案是 **循环队列** 🔥。

**循环队列**将存储数组逻辑上看成一个环——当 `rear` 或 `front` 到达数组末尾后，通过取模运算回到数组起点。比如 `rear = (rear + 1) % MAXSIZE`。这就带来了一个新的麻烦：**队空和队满的判定条件发生冲突**，因为两者都可能导致 `front == rear`。为解决这个问题，考研中有三种常用方法：

1. **牺牲一个存储单元** 🔥（最常用）：当 `(rear + 1) % MAXSIZE == front` 时认为队满，此时 `front == rear` 就是队空。这种方法始终有一个单元空着不用。
2. **增设 size 字段**：记录当前队列中的元素个数，`size == 0` 队空，`size == MAXSIZE` 队满。
3. **增设 tag 字段**：用 `tag` 标记上一次操作是入队（tag=1）还是出队（tag=0）。`front == rear && tag == 0` 队空，`front == rear && tag == 1` 队满。

**求队列长度**：`(rear - front + MAXSIZE) % MAXSIZE`，考卷上直接套公式。

> **考研出题套路**：给你当前 front、rear、MAXSIZE 的值，问队列长度或还能入队几个元素。核心就是那行取模公式，小心区分 `front` 和 `rear` 的初始值（有的约定 `front = 0, rear = 0`，入队时先放再移；有的约定 `front = 0, rear = 0`，入队时先移再放）。

### 队列的链式存储

链队 🔥 由一个带**头结点的单链表**表示，额外维护两个指针——`front` 始终指向头结点（不存数据），`rear` 指向队尾结点。入队在 `rear` 之后插入新结点并更新 `rear`，出队删除 `front->next` 所指结点。当队列中只剩最后一个元素时，出队后还需要把 `rear` 也指向头结点（否则 `rear` 就指向了一个已释放的结点）。链队不存在队满的问题，适合无法预知最大容量的场景。

> 链队在考研代码题中出镜率比链表本身低得多，但选择题中偶尔考察「链队入队/出队时哪些指针需要修改」。

### 双端队列

双端队列 🔥 允许在**两端都可以进行插入和删除**操作。如果限制只能一端插入、另一端删除，就退化回了普通队列。实际中有一个非常重要的变种叫**输出受限的双端队列**（一端可插可删，另一端只能插入），以及**输入受限的双端队列**（一端可插可删，另一端只能删除）。

考研中双端队列最经典的考法：**给你一个输入序列（如 1,2,3,4），问哪些输出序列可以通过某种受限双端队列得到**。解这类题的方法和出栈序列合法性判断非常类似——模拟每一种可能的操作序列，看能否得到目标输出。双端队列在考研中偶尔以选择题出现，难度不高。

> 🔥 双端队列在实际中极其常用——Java 的 `ArrayDeque` 和 Python 的 `collections.deque` 都是双端队列，用作栈和队列都比传统的 `Stack`/`Queue` 性能好。不过考研不考语言库的具体实现。

## 栈和队列的应用

### 栈在括号匹配中的应用

括号匹配 🔥 是栈最经典的应用场景之一，也是 408 选择题的常客。算法思想非常简单：从左到右扫描表达式，遇到左括号（`(`、`[`、`{`）入栈，遇到右括号时检查栈顶是否是对应的左括号——匹配就弹栈，不匹配或栈为空就报错。扫描结束后栈为空说明完全匹配。

这个算法三步走，时间复杂度 O(n)，栈的容量就是括号的最大嵌套深度。除了括号匹配外，同样的思路还可以用来检查 HTML/XML 标签是否正确闭合 🔥（但考研只考括号匹配）。

### 栈在表达式求值中的应用

表达式求值 🔥 是 408 中栈相关内容的最高频大题考点，几乎每 2–3 年就会在选择题或简答题中出现一次。考研主要考察三种形式之间的转换：

**中缀表达式**就是我们日常写的 `A + B * C`。因为运算符在操作数中间，人一读就知道先算 `B * C` 再加 `A`，但机器需要借助优先级规则才能确定计算顺序。

**后缀表达式（逆波兰式）** 🔥 是最实用的形式，运算符放在两个操作数之后：`A B C * +`。计算后缀表达式只需要一个栈——从左到右扫描，遇到操作数入栈，遇到运算符弹出两个操作数做运算再把结果入栈，最后栈中唯一的值就是结果。这个过程极其机械，编译器也正是用这种形式来计算表达式的。

**中缀转后缀**的经典算法如下：初始化一个运算符栈，从左到右扫描中缀表达式——操作数直接输出，遇到运算符时，如果栈顶运算符优先级 ≥ 当前运算符，就把栈顶弹出输出（不断重复），然后把当前运算符入栈。括号特殊处理：左括号直接入栈，右括号把栈中到左括号为止的运算符全部弹出输出。

**前缀表达式（波兰式）** 💀 在考研中出现频率远低于后缀。转换方式和中缀转后缀类似，只不过从右向左扫描，且优先级判断略有不同。实际中几乎没有系统使用前缀表达式，**考研了解基本概念即可，不会出计算大题**。

### 栈在递归中的应用

递归 🔥 的底层实现就是栈——操作系统中称为**系统栈**。每次递归调用，系统会把函数的局部变量、参数和返回地址压入系统栈，当递归开始返回时再逐一出栈恢复现场。这就是为什么递归层数太深会导致**栈溢出（Stack Overflow）**——因为系统栈的空间是有限的。

考研中学习「递归工作栈」的目的不在于让你去碰系统调用，而在于理解**递归转非递归**的手工方法。任何递归算法都可以用自己手工维护一个栈转换成非递归，这就是栈在算法设计中的核心价值。二叉树的非递归遍历就是典型例子：借助一个栈来模拟系统栈的行为，显式保存待处理的结点。

### 队列在层次遍历中的应用

层次遍历 🔥 是二叉树上一种自顶向下、从左到右的遍历方式，需要一个队列辅助：先将根结点入队，然后反复执行——出队一个结点、访问它、把它的左右孩子（如果有）入队，直到队列为空。二叉树的层次遍历在 408 中通常是「遍历」章节的一道选择题。

同样的层次遍历思想用在图上是 **广度优先搜索（BFS）** 🔥。BFS 是图论最基础的算法之一，408 的大题经常出现（最短路径、最小生成树相关计算都基于 BFS 思想）。操作系统中的磁盘调度算法、计算机网络中的路由算法也都有 BFS 的影子——这说明队列是一个非常基础又广泛应用的数据结构。

### 特殊矩阵的压缩存储

当矩阵中有大量相同元素（特别是 0 元素）或元素分布有规律时，为了节省存储空间，可以对矩阵进行压缩存储。

- **对称矩阵** 🔥：只有储下三角（含对角线）元素即可，上三角可以通过对称关系得出。n 阶对称矩阵只需 `n(n+1)/2` 个存储单元。地址计算公式是选择题的高频考点。
- **三角矩阵**（上三角/下三角）：对角线上方（或下方）元素全是常数 C。存储方式和对称矩阵类似，但需要额外一个单元存放常数 C。
- **三对角矩阵**（带状矩阵）⚠️：只有主对角线及其相邻的两条次对角线上有非零元素。地址计算需要代入公式 `k = 2i + j - 3`（或等价形式），考研中偶尔出现。

- **稀疏矩阵** 🔥：矩阵中非零元素极少（通常 < 5%）。稀疏矩阵不适用上面的规律性压缩方法，必须显式保存非零元素及其坐标。最常见的存储方式是**三元组**（行号, 列号, 值）和**十字链表**。考研只考三元组的基本概念，十字链表 💀 仅作了解。

> **考研出题套路**：给一个特殊矩阵和下标 i,j，问压缩存储后在一维数组中的下标 k 是多少。实质就是代入地址映射公式，考察下标从 0 还是从 1 开始——这是最容易被坑的地方，题目有时候不统一，务必看清题干。

## 常见题型

- **表达式转换与求值**（高频大题）：给中缀转后缀/前缀，或给后缀求值。手写栈的过程算一算，每一步的栈状态要能画出来。
- **循环队列判空/判满**（高频选择题）：三种方法各有判断条件，给 front、rear、MAXSIZE 问是否满/空/队列长度。
- **出栈序列的合法性判定**（高频选择题）：给定入栈序列（如 1,2,3，...，n）问某个序列是否合法。记住结论：出栈序列中，若某元素之后出现了编号更小的元素，这些更小元素必须按**递减顺序**出现。
- **出栈序列计数**：n 个元素的合法出栈序列总数 = **卡特兰数** C(n) = C(2n,n) / (n+1)。这个结论考卷上直接套公式即可，不需要推过程。
- **特殊矩阵存储地址计算**：对称矩阵/三对角矩阵的压缩存放下标映射公式是重点，注意区分按行优先还是按列优先，下标起始值是 0 还是 1。
