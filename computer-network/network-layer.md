# 网络层

> **考情提示**：网络层是计算机网络的核心章——子网划分与 CIDR 地址规划几乎每年一道大题（≈10 分），RIP 距离向量更新和 OSPF 最短路径也是必考内容。分值极高，是 408 网络得分的大动脉。

## 网络层的功能

网络层 🔥 是处理端到端数据传送的最底层——负责将分组从源主机跨越多个网络（路由器）送达目的主机。核心三大功能：

- **路由选择与分组转发** 🔥：根据路由协议计算出的路由表，决策每个分组该从哪个输出端口发往哪条下一跳链路。
- **异构网络互联** 🔥：用 IP 协议把所有不同物理网络的差异统一起来——让以太、Wi-Fi、光纤各种链路都互联为一个无缝的互联网。
- **拥塞控制**：当网络负载过重时主动削减注入分组速率——避免全网性能崩溃。TCP 的拥塞控制也是在这一层被辅助决策。

> 网络层向上只提供**无连接的尽力而为服务**——不保证定时送达、不保证顺序正确、不保证不丢失。这些保证由传输层（TCP）负责。

## 数据交换方式

### 电路交换

电路交换 💀 源于传统电话网：通信前必须**逐段建立一条独占的物理电路**直到对方，通信结束后释放。通话期间整条电路完全专用——时延稳定但带宽利用率极低，因为长时间空白也无人可用。不适用于突发性强的计算机数据。

### 报文交换

报文交换 💀 把整个消息作为整体一步一步传——路由器收到完整的报文后才找路由转发给下一跳（存储转发）。不需建立连接，但大报文会长时间占据缓冲区且端到端总时间特别慢（一个包必须完全到达才能发下一跳——无管线能力）。

### 分组交换

分组交换 🔥 🔥 是今天所有互联网的基础：将长报文在发端切成一系列小块分组，每个分组独立在路由间逐跳传递，发到目的地后重组排序。分组交换的极大优势是**管线化**——一个分组开始发送后几段时间后续分组陆续跟上，各分组在不同链路上并行传播，整体完成总时延远小于整个报文的存储转发。

分组交换进一步分两种方式：
- **虚电路** ⚠️：类似电路交换 + 分组层——发送前先建立虚电路路径，各路由器记住一条逻辑连接为同一流的分组标记虚电路号（VC）。分组的传达方向固定有序但无需独自独占链路。ATM/帧中继使用虚电路，现在已边缘化。
- **数据报** 🔥 🔥：每个分组独立携带完整的目的 IP 地址，各路由器各自查表独立转发不用维护连接状态——Internet 的数据服务就是基于数据报（IP 协议）。

### 三种交换的对比

| 对比项 | 电路交换 | 报文交换 | 分组交换（数据报） |
|--------|----------|----------|-----------------|
| 需建立连接 | 是 | 否 | 否 |
| 传输方式 | 实时比特流 | 存储转发整报文 | 存储转发分组 |
| 资源占用 | 独占 | 共享 | 共享 |
| 管线化并行 | 否 | 否 | 是 🔥 |
| 实时性 | 极高 | 差 | 中等 |
| 代表 | 历史电话网 | 历史电报 | **Internet** 🔥 |

## IPv4

### IPv4 地址

IPv4 地址 🔥 是全球互联网最基础的逻辑地址——32 位二进制数，通常用点分十进制表示（如 192.168.1.1）。

**分类 IP 地址（有类编址）** 🔥（考研必须熟知各类的起止范围）：

| 类 | 首字节范围 | 网络号位 | 默认掩码 | 特点 |
|:--:|-----------|:-----:|------|------|
| A | 1‒126 | 8 位 | /8 | 超大网络数少，每个网络可接上千万台 —  💀 已不单独分配 |
| B | 128‒191 | 16 位 | /16 | 中等网络 — 💀 已停单独分发 |
| C | 192‒223 | 24 位 | /24 | 小网 — 每个网 254 台最多 |
| D | 224‒239 | — | — | 多播地址 |
| E | 240‒255 | — | — | 保留 |

**特殊 IP 地址** 🔥（408 年年必考辨析）：
- **网络号全 0 + 主机号全 0**：本网络本主机（作为源，出发时不知道本点 IP）。
- **网络号 + 主机号全 0**：表示网络本身（如 192.168.1.0）。
- **网络号 + 主机号全 1**：广播到网络内所有主机。
- **127.x.x.x**：环回地址 🔥——发给 127.0.0.1 不进入网络直接被操作系统在本机返回——用于测试和本地通信。
- **0.0.0.0**：表示不确定的 IP（如 DHCP 分配前）。

**私有 IP 地址** 🔥：只可在内部网络使用，公网上路由器不转发；
- A 类私有：10.0.0.0/8
- B 类私有：172.16.0.0/12
- C 类私有：192.168.0.0/16

### 子网划分

子网划分 🔥 🔥 是在一个 IP 网络号内部通过**子网掩码**分割出更多细粒度的子网络——把主机号的高几位借来做**子网号**。子网掩码形式与 IP 相同（如 /26 = 255.255.255.192）。

核心套路计算（大分送分题）：
- **子网地址** = 主机 IP **按位与** 子网掩码
- **可用主机数** = 2^(32 − 掩码位数) − 2（去除网络地址和广播地址）
- **子网数** = 2^(借位数)

> 408 的大分计算题：给一个网段 192.168.1.0/24，要求再切成 4 段子网——需要向主机位借 2 位做子网位（2²=4），新的子网掩码 = /26，每个子网最多 62 台。然后列出各个子网的网络地址、可用 IP 和广播地址。

### CIDR（无分类编址）

CIDR（Classless Inter-Domain Routing）🔥 🔥 摒弃 ABD 类的旧限制——IP 地址只以「前缀 /n」记，如 10.0.0.0/23。任意数目的连续地址可以聚合成一个**路由聚合**项（也叫超网或路由聚合），大幅度降低路由器路由表项数量。

**最长前缀匹配** 🔥：当一个数据报的 IP 在路由表中匹配多个路由项时（因为聚合后前缀重叠），路由器选择**前缀最长的**（具有更具体的匹配子网）——这个是所有的路由查找的基本法则。

### ARP 协议（地址解析协议）

ARP（Address Resolution Protocol）🔥 🔥 是 IP 地址适配到下层 MAC 地址的关键协议。当一台主机知道目标 IP 但未知目标 MAC 地址时，它在局域网广播一个 ARP 请求（「谁是 192.168.1.1？」），目标主机识别到属于自己 IP 的单播应答 MAC 返回。整个过程透明进行，结果**在 ARP 缓存中保留几分钟**减少后续广播。

### DHCP 协议（动态主机配置）

DHCP（Dynamic Host Configuration Protocol）🔥 🔥 让主机在接入网络时自动获取 IP 地址和 DNS、网关等配置。正常流程四步：① 客户端广播 **DHCP Discover** ② DHCP 服务器回应 **DHCP Offer**（含分配的 IP 和租期）③ 客户端选择一项发出 **DHCP Request** ④ 服务器确认 **DHCP ACK**，客户端以此 IP 使用直到租期到来。到期半段会试图续约。

### ICMP 协议（网际控制报文）

ICMP 🔥 是 IP 层的控制辅助协议——用于报告路由途中的错误和诊断信息——运行在 IP 里面，封包在 IP 数据报中。

**ICMP 差错报告报文**：
- **终点不可达**：路由器或目的地无法将分组投出（网络、主机、协议、端口任何一个不可达情况）
- **时间超过（TTL 到期）**：分组限制了每次路由器减一跳 TTL，TTL 减为 0 即丢弃并回传 ICMP 时间超过——Traceroute 工具正是基于定时递增 TTL 的值探测路径
- **参数问题**：首部字段错误
- **改变路由（重定向）**：路由器建议发送方使用更好的下一跳路径

**ICMP 询问报文**：
- **PING（回送请求和应答）** 🔥：测试两台机器间连通性和往返延迟的极常用工具——PING 会发送回送请求，收到方回传回送应答。
- **时间戳请求/应答** 少量用于时钟同步。

## IPv6

### IPv6 的特点

IPv6 🔥 🔥 是因特网工程组织为取代 IPv4 地址枯竭设计的全新协议，采用 **128 位地址**（为地球所有电子设备每粒沙子分给数十个 IP）。IPv6 增强了以下核心优势：极大地址空间、**简化基本头部**（去除网际校验和与分片字段存在等不用功能的默认首部）——由扩展首部替代可选信息（比 IPv4 的高效紧凑更具设备即插即用即发即通）；内置 IPSec 支持端到端的通信安全；支持设备无服务器干预的自配地址（SLAAC 自动地址）。目前全球 IPv6 部署已过半。

### IPv6 地址表示

IPv6 地址用**冒号十六进制**八组每四位十六进制表示，如 `2001:0db8:0000:0000:0000:ff00:0042:8329`。为了可读性有**零压缩**的规则：① 每组前导 0 可省略（如 `:0042:` → `:42:`）② 全零的连续组用 `::` 一次性省略（但每个地址只能一次 `::` 以避免歧义）。上面的地址压缩后为 `2001:db8::ff00:42:8329`。

### IPv6 与 IPv4 的过渡

两种主流过渡共存策略：
- **双栈协议** 🔥：设备同时运行 IPv4 和 IPv6 双协议栈。若通信的双方都支持即用 IPv6，对方仅支持 IPv4 则回退到 IPv4——目前通用过渡期做法。
- **隧道技术** 🔥：当两个 IPv6 网络被中间 IPv4 区域隔断或反过来——把 IPv6 分组封装在 IPv4 数据报中穿越 IPv4 区域过去（6to4 隧道）——出口再解封出 IPv6 分组送达目标。是早期跨异构网最常用的过渡方案。

## 路由协议

### 自治系统（AS）

互联网上按行政管理域和路由政策划分的独立网络实体叫做**自治系统（AS）** 🔥。每个 AS 选一个 16/32 位 AS 号码代表。内部的路由协议叫做**内部网关协议（IGP）**，AS 之间的路由协议叫做**外部网关协议（EGP）**。

### 内部网关协议（IGP）

#### RIP（距离向量路由）

RIP（Routing Information Protocol）⚠️ 基于经典的 Bellman-Ford 距离向量算法——每路由器只与直接相连的邻居路由器交换其所有距离向量（到各目的网络的距离）。度量标准为**跳数**，每经过一个路由器加一。RIP 有显著的缺点：

- **慢收敛问题** 🔥：整个网络拓扑改变的消息要逐跳扩散——某个网络如果断开会引发「坏消息传得慢」甚至短期环路。
- **最大跳数 15** 🔥：不可逾越 16 = ∞（不可达），因此强烈限制大型网络部署。

> 💀 RIP 在很多实际网络已过时，被 OSPF 取代。408 热衷考 RIP 距离向量的更新模拟，经常给一网格拓扑和路由表让你填多轮迭代后各路由表的变化。

#### OSPF（链路状态路由）

OSPF（Open Shortest Path First）🔥 🔥 是目前最主流的内网路由协议。不使用距离向量而改用**链路状态**——每路由器知道整个互联区的完整拓扑图，基于 Dijkstra 最短路径算法独立自算出到各网段的最优路径。链路状态变化时洪泛更新传播迅速且一致。OSPF 划分了**区域（area）** 包括**骨干区域 area 0** 与其他非骨干区域连接多区拓扑便于大规模网络的分级部署。OSPF 使用 Hello 协议持续互存活邻接维持同步。

### 外部网关协议（EGP）

#### BGP（边界网关协议）

**BGP（Border Gateway Protocol）** 🔥 🔥 是如今全球化互联网选路的真正核心——承担各 AS 间的可策略路由选择。BGP 依靠**路径向量**——不仅知道的距离——BGP 每一路由项包含通向目标网络的 AS 路径序列表。各 AS 的策略（经济政策、性能政治意愿）如何选路决定出口。BGP 会话运行在 TCP 端口 179 可靠传递大量路由更新。

### 三种路由协议的对比

| 协议 | 类型 | 算法 | 度量 | 封装协议 | 实际状态 |
|------|------|------|------|----------|:--:|
| RIP | IGP | 距离-向量 | 跳数 | UDP | ⚠️ 过时 |
| OSPF | IGP | 链路状态 + Dijkstra | 带宽/成本 | IP | 🔥 当前主流 |
| BGP | EGP | 路径-向量 | 策略/路径属性 | TCP | 🔥 全球骨干 |

## IP 多播

### 多播地址

IP 多播（Multicast） 🔥 发送单个分组而能同时把数据投递到一组属于同一个**多播组**的目标主机——用于直播/视频会议/股市 ticker 等。多播地址使用 D 类 IPv4 空间（224.0.0.0‒239.255.255.255）。

### IGMP 协议

**IGMP（Internet Group Management Protocol）** 是主机与局域网上一层路由器间使用的管理协议——通过 IGMP 主机可动态地**加入/离开**多播组。路由器依赖 IGMP 知道本局网内有哪一组成员是否存在决定是否继续转发多播流量到此端口。

## 网络层设备

### 路由器

路由器 🔥 🔥 是网络层工作核心设备——在每个 IP 分组到来的入接口时，硬件提取目的 IP、拆去下层帧首包装，经转发表（由路由协议生成）匹配「下一跳 IP」连同转发到合适的出口接口重新装帧发出。路由器没有广播域，每个端口分别是一个独立的物理网络广播区域。

分组转发的三个关键步骤（大分选择/简答题中画出流程图）：
1. **查找路由表**（最优前缀匹配——最近用的路由与目的 IP 掩码与比较比较）。
2. **修改 TTL** 字段将 TTL 减一。如果 TTL = 0，则丢弃分组并送 ICMP 反馈。
3. **数据链路层封装**：如果下一跳 IP 对应局网的目标 MAC 已经通过 ARP 获得——重新装载到新局网帧送出。如果没有 ARP 则触发 ARP 请求等待再发。

## 常见题型

- **子网划分和 CIDR 的地址规划**（大题必考）：划分多个子网中每子网的网络地址、掩码、可用 IP 范围、广播地址全部列齐。
- **RIP 距离向量更新的模拟**：给初始各路由器和邻接网段——迭代到各路由器与网络拓扑平稳后写出最终距离向量的路由表。
- **OSPF 最短路径计算**：给各路由器-链路的权重表——通过 Dijkstra 手工计算从给定源到各段的最短开销并输出最优邻接导出列表。
- **路由器转发流程（路由表匹配）**：给一个简化的路由表条目和多条目的多个输入分组 IP → 选出下一跳和出口接口——重点是匹配最长前缀。
