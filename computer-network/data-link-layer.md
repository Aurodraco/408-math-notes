# 数据链路层

> **考情提示**：数据链路层是 408 网络中分值最高的一章——CSMA/CD 最短帧长计算、GBN/SR 窗口计算、交换机自学习是三大核心考点。通常 2 道选择题（4 分）加可能一道大题中的多个子问题。逻辑推理多、计算套路固定——练熟套路上场即可。

## 数据链路层的功能

数据链路层 🔥 负责**同一直接相连网络中的两个相邻节点**间可靠/不可靠的帧发送——工作于物理层之上以帧为单位处理。其主要服务模式有三种：

- **无确认的无连接服务** 🔥：每帧直接发送出去不需要接收方给确认——效率高，适用于误码率低的线路。以太网默认即是此类。
- **有确认的无连接服务**：每一帧独立编号、接收方需返回确认但无需连接建立——无线网络常用之以抗高误码。
- **有确认的面向连接服务**：建立连接 → 传送帧 → 释放连接；在传输期间保障每帧正确无误到达且有顺序保持。

该层还负责：**链路管理**、**帧定界**（标识一帧的开始和结束）、**帧同步**、**透明传输**（保证数据部分不会意外出现帧界标志符）、**流量控制**（限制发送方不要淹没慢接收方）和**差错控制**。

## 组帧

要让接收方在比流中分辨每帧从哪开始到哪结束，有四种组帧方法：

- **字符计数法** 💀：在帧头加一个字节记录本帧的字节总数。简单但计数一旦出错会对后面全部帧的定位连锁崩溃——不安全。
- **字符填充的首尾定界法** 🔥：用特殊字符标识帧首和帧尾（如 DLE STX / DLE ETX）。若数据中出现与首尾符号相同的字节，则在其之前加一个转义字符（DLE）进行填充——类似于字符串转义处理。
- **比特填充的首尾定界法（零比特填充）** 🔥 🔥：用帧标志 01111110 标识帧的开始和结束——发送方在数据中每遇到连续 5 个「1」就自动插入一个「0」避免产生假的标志。接收方看到连续 5 个「1」后根据下一个「0」抹除填充恢复原数据——**HDLC 和 PPP 均使用零比特填充法** 🔥。
- **违规编码法** 💀：利用物理层码规则中保留不用的信号编码表示帧开始与结束。现多数被比特填充法替代。

## 差错控制

### 检错编码

- **奇偶校验码** 🔥：在数据位之后附加一个奇校验位/偶校验位使整个帧中 1 的总数符合奇数/偶数。只能检单比特错，实现极为简单。用于检测暂时内存错误或低误码场合。
- **循环冗余校验（CRC）** 🔥 🔥：在组成原理中已详述原理——此处重点是 CRC 是**数据链路层**最普遍采用的检错技术——**每种以太网帧的 FCS 字段就存放 CRC 结果**。该层所选用生成多项式已标准化，固定 CRC-32 为 <math>G(x)</math> 供全以太网统一检错。

### 纠错编码

- **海明码** 🔥：让接收方不但知道有错，还能定出哪一位出错并自我纠正。海明码原理见组成原理章，数据链路层中使用海明码的场景较少，仅在误码率极高且需要**采用前向纠错（FEC）** 的卫星/深空通信中使用。

> 408 的 CRC 计算常在该层出现——给数据序列和生成多项式，推 CRC 余数并输出全发送序列。

## 流量控制与可靠传输

### 停止-等待协议

停止-等待协议 🔥 是最原始、最简单的可靠传输协议——发送方发一个帧后必须等待对方返回**确认帧（ACK）** 才能发下一帧。如果定时器超时没有收到 ACK 则自动重发。接收方按收到帧的编号（1 位序号 0/1 交替）判断是否重复帧。信道利用率极低——发送方发完一个帧后大部分时间在「傻等 ACK」。

信道利用率公式：`U = (t_send) / (t_send + 2×t_prop + t_ack)`，如果传播延时较大而帧短小则利用率极低。

### 后退 N 帧协议（GBN）

后退 N 帧协议（Go-Back-N）🔥 🔥 通过「流水线」提升利用率——发送方允许连续发送多个帧而不必等待 ACK。接收方采用**累积确认**——ACK n 表示 n 号及之前的帧全都正确收到。一旦发生超时，发送方**倒退到丢帧的位置**把该帧以及其后随之发过的帧全部重发——因此出现不必要的大量重复传输。

GBN 的发送窗口大小 W_T 🔥 必须满足**1 ≤ W_T ≤ 2ⁿ − 1**（n 为序号位）。接收窗口 W_R = 1（一次只收期望的下一帧，乱序直接丢弃）。

### 选择重传协议（SR）

选择重传协议（SR）🔥 🔥 是 GBN 的高效改进——接收方也有多个帧的缓冲窗口，乱序到达的正确帧会被**暂时缓存**而不丢弃。发送方只需要重发那个真正丢失的帧（但不是逐帧确认的一味确认全部）。确认采用**逐帧确认**而非累计确认。

SR 双重重要条件 🔥：发送窗口 W_T 和接收窗口 W_R 满足**W_T = W_R ≤ 2ⁿ⁻¹**（为序号位数的半数以下）。若超过此值序号循环将不可区分新旧帧造成灾难性混乱。

### 三种协议的对比

| 协议 | 窗口 | 确认类型 | 乱序缓冲 | 重传策略 | 信道利用率 |
|------|------|----------|:--:|------|:--:|
| 停止-等待 | W_T = 1, W_R = 1 | 逐帧确认 | 无 | 发一帧等确认超时则重发 | 极低 |
| GBN | 1 ≤ W_T ≤ 2ⁿ−1, W_R = 1 | 累积确认 | 无 | 超时重传整个窗口 | 中 |
| SR | W_T = W_R ≤ 2ⁿ⁻¹ | 逐帧确认 | 有 | 只重传丢帧 | 高 |
