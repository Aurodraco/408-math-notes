# 应用层

## 网络应用模型

### 客户/服务器模型（C/S）

- 客户是服务请求方，服务器是服务提供方
- 服务器持续运行，客户间歇连接

### P2P 模型（对等模型）

- 每台主机既是客户又是服务器
- 无需集中式服务器
- 可扩展性好

## 域名系统（DNS）

### 域名空间

- 顶级域名：.com、.org、.net、.cn 等
- 二级域名、三级域名

### 域名服务器

- 根域名服务器
- 顶级域名服务器
- 权限域名服务器
- 本地域名服务器

### 域名解析过程

- 递归查询
- 迭代查询
- DNS 缓存

## FTP（文件传输协议）

### FTP 的工作原理

- 控制连接（端口 21）
- 数据连接（端口 20）
- 主动模式和被动模式

## 电子邮件

### 电子邮件系统的组成

- 用户代理（UA）
- 邮件服务器
- 邮件协议

### SMTP（简单邮件传输）

- 发送邮件，端口 25
- 只传 ASCII 码

### POP3 / IMAP

- POP3：下载后从服务器删除（端口 110）
- IMAP：邮件保留在服务器（端口 143）

### MIME（多用途互联网邮件扩展）

- 支持非 ASCII 数据
- 邮件加密（S/MIME、PGP）

## 万维网（WWW）

### 万维网的概念

- 统一资源定位符（URL）
- 超文本传输协议（HTTP）

### HTTP 协议

- HTTP 1.0 vs 1.1 vs 2 vs 3
- HTTP 报文格式（请求报文、响应报文）
- 持久连接（Keep-Alive）和非持久连接
- HTTP 无状态性

#### HTTP 方法

- GET、POST、HEAD、PUT、DELETE、OPTIONS

#### HTTP 状态码

- 1xx：信息
- 2xx：成功（200 OK）
- 3xx：重定向（301 永久、302 临时）
- 4xx：客户端错误（404 Not Found）
- 5xx：服务器错误（500 Internal Server Error）

#### Cookie

- 弥补 HTTP 无状态性
- 会话跟踪

## 常见题型

- DNS 的查询过程
- HTTP 的持久连接与非持久连接（RTT 计算）
- FTP 控制连接与数据连接的关系
- SMTP 与 POP3/IMAP 的区别
