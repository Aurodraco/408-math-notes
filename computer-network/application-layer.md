# 应用层

> **考情提示**：应用层是计算机网络的终章，408 中一般 1 道选择题（2 分）。重点：DNS 查询过程（递归 vs 迭代）、HTTP 持久/非持久连接的 RTT 计算、FTP 的控制/数据双连接模型。概念多但每个都浅——记熟协议端口号和原理即可。

## 网络应用模型

### 客户/服务器模型（C/S）

C/S 模型 🔥 是最传统也是最主要的网络应用架构——**服务器**是服务提供方，拥有固定的 IP 地址和端口，持续运行等待连接；**客户**是服务请求方，间歇连接服务器获取服务。客户和客户之间不直接通信。典型实例：Web 服务器（HTTP）、FTP 服务器、邮件服务器。优点是易于集中管理和安全控制；缺点是随着客户量增加服务器会成为性能瓶颈。

### P2P 模型（对等模型）

P2P 模型 🔥 打破了服务器的绝对中心角色——**每个节点既是客户又是服务器**，直接和其他节点共享资源而不经过集中服务器。P2P 最大的优势是**可扩展性**——节点越多系统的总带宽和服务容量越大（用户贡献自己的存储和上传带宽）。典型的如 BitTorrent、区块链节点、Skype 早期的流传输。缺点在于安全性难控制和缺乏集中认证。

> 408 选择题高频：C/S 可集中管控，P2P 扩展性好但难以管理。二者对比属于送分题。

## 域名系统（DNS）

### 域名空间

DNS 🔥 🔥 是互联网的命名基础设施——把人类可读层次化域名（如 `www.example.com`）翻译成机器可路由的 IP 地址。域名层级从右向左增加：**根域（.）** → **顶级域名**（.com / .org / .cn / .edu）→ **二级域名**（如 example）→ **三级域名**（如 www）。上层域名授权下层子域自行控制——分散了 DNS 数据库的管理压力。

### 域名服务器

分布在世界各地的 DNS 服务器按职责分四层：

- **根域名服务器** 🔥：全球 13 组逻辑根服务器——存储所有顶级域的 DNS 服务器地址——找到顶级域的名字服务器的第一入口。
- **顶级域名服务器（TLD）**：负责对应自身顶级域（如 .com 域）下的权威服务器地址。
- **权限域名服务器** 🔥：某个具体域（如 example.com）的实际资源记录所在服务器——给出这个域里面所有主机域名的最终 IP 回答。
- **本地域名服务器** 🔥：最靠近用户的那台 DNS 服务器（一般为运营商或机构提供的缺省服务器）——缓存大多数查询结果以大幅减小上游查询量。

### 域名解析过程

域名解析 🔥 分为两种模式：

- **递归查询**：客户向本地 DNS 服务器发起查询，本地服务器得不到回答就代劳向下查询直到获得权威答案并返回。客户端只问了一次。一般客户 → 本地都用递归。
- **迭代查询** 🔥：自己一级一级问下家——本地 DNS 服务器向根服务器发出请求，根返回 TLD 地址告诉它「你去问 .com 服务器」——本地再自己去问 TLD 获取权威服务器地址继续问权威。各步骤需多次交互但避免上层代理压力积累。

**DNS 缓存** 🔥：所有层级服务器都可以缓存最近查询的结果一段 DNS 配置的 TTL（Time To Live）时间——大幅减少重复查询的流量。

## FTP（文件传输协议）

### FTP 的工作原理

FTP 🔥 使用**双连接模型**——这是它在应用层最独特的特征：

- **控制连接** 🔥：FTP 客户端连到服务器的端口 21，在整个 FTP 会话期间一直保持打开——用户登录、改变目录、请求文件列表等命令通过此连接传输。
- **数据连接** 🔥：在实际传输一个文件或目录列表数据时，另外动态建立一条数据连接——过去默认通过服务器端口 20 连接到客户端的一个随机端口（主动模式）。现在的工业标准趋于被动模式（PASV）：在服务器的随机端口上接受客户端的主动数据连接——更适合穿越防火墙和 NAT。

> 408 常见考题：「FTP 使用几个 TCP 连接？」——两个。「控制连接和数据连接哪个是一直保持的？」——控制连接。

## 电子邮件

### 电子邮件系统的组成

电子邮件系统 🔥 由三部分构成：**用户代理（UA）**（你用来收发邮件的软件——如 Outlook、Gmail 网页）、**邮件服务器**（收发邮件的中转邮局）、**邮件发送/接收协议**（SMTP 发、POP3/IMAP 收）。

### SMTP（简单邮件传输）

SMTP 🔥 负责把邮件从发送方的邮件服务器投递到接收方的邮件服务器（TCP 端口 25）。它只传送 ASCII 文本——非 ASCII 的附件和中文需要由 MIME 额外编码。SMTP 采用命令/响应式对话，用 EHLO、MAIL FROM、RCPT TO、DATA 一系列命令传送。

> 注意：SMTP 是「推」协议，把邮件推向目的服务器。它不是用来收邮件的。

### POP3 / IMAP

用户取得自己邮件时要用到**读信协议**：

- **POP3（Post Office Protocol v3）** 🔥：将邮件从邮件服务器**下载到本地客户端后自动从服务器删除**。端口 110。对于多设备的用户不友好——下载后另一设备看不到该邮件。
- **IMAP（Internet Message Access Protocol）** 🔥 🔥：邮件**始终保留在服务器上**，不同设备只是服务器邮箱的镜像浏览——同一封邮件在手机和电脑上同步。端口 143。当前是目前的主流。

### MIME（多用途互联网邮件扩展）

MIME 🔥 是为了克服 SMTP 只能传 ASCII 的限制而扩展的标准——支持非英语文字（中文、日文）和附件二进制文件编码后仍以 ASCII 流嵌入邮件——对应格式包含 Base64 等编码方式，还可为邮件赋予加密和数字签名功能（S/MIME、PGP）。

## 万维网（WWW）

### 万维网的概念

万维网（World Wide Web）🔥 基于 HTTP 协议的全球分布的信息网络——三个核心概念：**URL**（统一资源定位符，标识资源位置）、**HTTP**（超文本传送协议，规定客户端与服务器间如何请求与响应资源）和 **HTML**（超文本标记语言，网页内容的呈现格式）。

### HTTP 协议

HTTP 🔥 🔥 是 Web 最基础的通信协议，每个 Web 访问都至少一次 HTTP 请求-响应交互：

- **HTTP 1.0**：默认**非持久连接**——每次请求一个资源都新建一个 TCP 连接，拿到响应后就关闭。简单但每访问一个图片或 CSS 文件都是一次 TCP 连接和挥手费时。💀 过时。
- **HTTP 1.1** 🔥：默认**持久连接**（Keep-Alive）——同一次 TCP 连接上可连续发多个请求和接收多个响应，大幅减少 TCP 握手和慢开始的开销。现在还广泛使用。
- **HTTP/2** 🔥：在 1.1 基础上添加流的多路复用——一个连接内并行传输多个独立请求组合更高吞吐——现代网站标准。
- **HTTP/3** 🔥：彻底不与 TCP 绑定而独立在 QUIC（UDP 上的连接协议）层运行——减少连接建立延迟和头部阻塞——最新安全高效——Google、Facebook 等服务快速迁移。

**HTTP 无状态性** 🔥：HTTP 本身不保存之前请求的任何信息——连续两次请求的上下文无保留。所以需要 Cookie 来在客户端维护会话状态。

#### HTTP 方法

- **GET** 🔥：请求指定资源——无请求体，幂等。由 URL 参数传递数据。
- **POST** 🔥：向服务器提交需要处理的数据（表单上递、上传等）。有请求体。非幂等。
- **HEAD**：同 GET 但不返回实际数据——只获取响应头，用于检查资源存在性。
- **PUT**：上传内容到指定 URL 完整替代原资源——常用于 REST API 更新。
- **DELETE**：删除指定 URL 资源。 
- **OPTIONS**：询问服务器支持哪些方法及跨域请求预检。

#### HTTP 状态码

| 码类 | 含义 | 典型例子 |
|:--:|------|------|
| 1xx | 信息型 | 100 Continue |
| 2xx | 成功 | **200 OK**（最常见）|
| 3xx | 重定向 | 301 永久移动 / 302（临时）|
| 4xx | 客户端错误 | **404 Not Found**（资源不存在）、**403 Forbidden** |
| 5xx | 服务器错误 | 500 Internal Server Error / 503 Service Unavailable |

#### Cookie

**Cookie** 🔥 是服务器要求浏览器在本地存储一小段键值文本的机制。下一次访问相同域时浏览器自动附上这些 Cookie——以此跟踪用户会话，弥补 HTTP 无状态性的缺乏。Cookie 广泛用于登录状态保持、购物车信息、跟踪分析等。但存在隐私滥用和 CSRF 攻击风险，现在网络标准在迁移向更加安全的令牌机制。

## 常见题型

- **DNS 的查询过程**：给域名层次图→说明迭代与递归查询分别走哪几个环节，需访问哪几级服务器。
- **HTTP 的持久连接与非持久连接（RTT 计算）**：给网页所含对象数，分别计算非持久和持久连接下的总传送结束所需的 RTT 数。非持久逐个对象每个 TCP 连接一个 RTT（需加延两端 TCP 建立和网络时延）；持久化连接只需再加并行取的时间。
- **FTP 控制连接与数据连接的关系**：牢记双连接模型——端口 21 控制数保持、端口 20 或随机端口供数据临时建立。
- **SMTP 与 POP3/IMAP 的区别**：SMTP=发邮件推送；POP3/IMAP=取邮件。POP3 下回就删 IMAP 留在服务器。
