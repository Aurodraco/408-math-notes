# 传输层

## 传输层概述

### 传输层的功能

- 提供进程之间的逻辑通信（端到端）
- 复用和分用
- 差错检测

### 套接字（Socket）

- IP 地址 + 端口号
- 标识网络中的唯一进程

## UDP 协议

### UDP 的特点

- 无连接
- 不可靠（尽力交付）
- 面向报文
- 没有拥塞控制
- 首部开销小（8 字节）
- 支持一对一、一对多、多对一、多对多

### UDP 的应用场景

- DNS、SNMP、RIP、多媒体通信

## TCP 协议

### TCP 的特点

- 面向连接
- 只支持一对一
- 可靠传输（确认 + 超时重传）
- 全双工通信
- 面向字节流

### TCP 报文段格式

- 源端口、目的端口、序号、确认号
- 数据偏移、保留位、标志位（URG/ACK/PSH/RST/SYN/FIN）
- 窗口、校验和、紧急指针、选项

### TCP 连接管理

#### 三次握手

- 第一步：SYN = 1, seq = x
- 第二步：SYN = 1, ACK = 1, seq = y, ack = x + 1
- 第三步：ACK = 1, seq = x + 1, ack = y + 1

#### 四次挥手

- 第一步：FIN = 1, seq = u
- 第二步：ACK = 1, seq = v, ack = u + 1
- 第三步：FIN = 1, ACK = 1, seq = w, ack = u + 1
- 第四步：ACK = 1, seq = u + 1, ack = w + 1

### TCP 可靠传输

- 序号
- 确认（累积确认）
- 超时重传（自适应超时时间计算）
- 冗余 ACK 和快速重传

### TCP 流量控制

- 滑动窗口协议
- 接收方窗口（rwnd）告知发送方剩余缓存
- 零窗口探测

### TCP 拥塞控制

#### 慢开始

- cwnd 从 1 开始，每 RTT 翻倍
- 达到慢开始门限（ssthresh）进入拥塞避免

#### 拥塞避免

- cwnd 每 RTT 线性增加 1

#### 快重传

- 收到 3 个重复 ACK，立即重传丢失报文

#### 快恢复

- ssthresh = cwnd / 2
- cwnd = ssthresh，直接进入拥塞避免

#### TCP Tahoe vs TCP Reno

### TCP 计时器

- 重传计时器
- 持续计时器（用于防止零窗口死锁）
- 保活计时器
- TIME_WAIT 计时器

## 常见题型

- TCP 连接建立和释放的序号分析（大题重点）
- 拥塞控制窗口变化曲线图
- TCP 往返时间和超时时间计算
- UDP 校验和计算
