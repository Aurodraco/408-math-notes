# 计算机网络概述

> **考情提示**：本章属于入门概述，408 中一般 1 道选择题（2 分）。重点：时延计算（发送 vs 传播）、分组交换 vs 电路交换的对比。概念极其精炼——分清四个时延的类型含义即可满分。

## 计算机网络的基本概念

### 计算机网络的定义

计算机网络 🔥 是利用通信设备和线路将地理上分散的多个自治计算机系统互连起来，按照网络协议进行数据通信，实现**资源共享**和信息传递的系统。关键词有两个——计算机网络的基本目标是**数据通信**（两台计算机之间传送比特流），最终目标是**资源共享**（共享硬件、软件、数据资源）。

### 计算机网络的组成

从三个维度看网络的组成：

- **物理组成**：计算机网络 = 硬件（主机、通信线路、交换机、路由器等）+ 软件（协议栈、网络操作系统）+ **协议**（规约通信双方对话的规则集合）。
- **工作方式** 🔥：网络分为**边缘部分**（用户直接使用的主机和终端设备）和**核心部分**（由大量路由器构成的通信骨架——负责高效转发分组）。边缘设备是「网络的用户」，核心部分是「网络的承运商」。
- **功能组成**：**通信子网** 🔥（负责数据传输——路由选择、流量控制、差错控制；由交换机、路由器、通信线路组成）和**资源子网**（负责数据处理——提供资源的计算机、终端和服务器）。

> 408 选择题常见考点：「路由器和交换机属于通信子网还是资源子网？」——通信子网。

### 计算机网络的功能

四大核心功能 🔥：
- **数据通信**：计算机之间互相传输数据——网络存在的最基本理由。
- **资源共享**：共享远处的打印机、存储设备、计算能力、软件和数据。
- **分布处理**：将大的任务分散到网络上的多台计算机协同完成——负载均衡、分布式计算。
- **提高可靠性**：多台计算机网上互为备份——一台故障后可由其他继续服务（容错和冗余）。

### 计算机网络的分类

- **按分布范围** 🔥：**个域网 PAN**（Bluetooth/红外）、**局域网 LAN** 🔥（楼/校园内——以太网，最高见）、**城域网 MAN**（覆盖城市）、**广域网 WAN**（跨国家）。
- **按交换技术** 🔥：**电路交换**（通话期间建立独占电路—电话网）、**报文交换**（整个报文逐跳转发—历史上的电报网 💀）、**分组交换** 🔥🔥（拆分报文成多个分组——每个分组独立走路径到达后再重组——互联网的基础）。
- **按传输介质**：有线网（双绞线/光纤）和无线网（Wi-Fi/蜂窝网络）。

> 408 必考分辨图：「分组交换比报文交换更灵活效率高在哪？」答案——分组可以管线服务（流水传送），一个分组一边到一边就开始传而不用等完整报文收全。

## 计算机网络的性能指标

### 速率、带宽、吞吐量

- **速率** 🔥：数据的传输率——单位时间传送的比特数（bps）。
- **带宽** 🔥：在通信工程中本意是信道通过的频率范围（赫兹），但在计算机网络中已经等同于**最大可达速率**（bps）——指信道能支持的最高比特率。
- **吞吐量** 🔥：实际成功的有效数据传送速率（受拥塞、重传开销等影响——对真实网络的度量值）。

### 时延

时延（延迟 / Latency）是数据从一端传送到另一端所花的总时间，由**四个组成部分累加** 🔥 🔥（408 年年必考）计算：

1. **发送时延（传输时延）**：分组从第一个比特发生至最后一个比特发完的时间。**发送时延 = 数据长度 / 发送速率**。取决于网络接口卡的速率和分组大小。
2. **传播时延** 🔥：电磁波信号在信道中从一个端口传播到对端口的时间。**传播时延 = 信道长度 / 信号传播速度**（光纤/铜线约 2×10⁸ m/s）。与发送时延完全无关——二者混淆是 408 的大忌！
3. **处理时延**：路由器解析分组首部、做差错校验和路由选择所花的时间——通常极短。
4. **排队时延**：分组在路由器的输入/输出队列中等待被处理的排队时间——网络拥塞时的最大变量时延。

> 408 选择题千篇一律：「一个 1000 字节分组在 100Mbps 链路上发送时延」 = 1000×8 / 100×10⁶ = 80 μs。「同一条 20km 光纤传播时延」≈ 20km / (2×10⁸ m/s) = 0.1 ms。问总时间 = 上面两项相加。区分不了这两个分量的含义直接丢分。

### 时延带宽积

**时延带宽积 = 传播时延 × 带宽** 🔥——代表第一个比特将到达远端时发送端共已发送的比特数。也就是管道中可能同时在路上的比特容量。这个数值大表示链路「长而粗」——需更大的发送窗口才能填满管道充分利用链路。

### 往返时延（RTT）

**RTT（Round-Trip Time）** 🔥 是指从发送端发送一个比特（或分组）到接收端、接收端收到后立即发回复（即应答）到达发送端的双向总时间。TCP 的超时重传定时器就是根据 RTT 估算的。RTT ≈ 2 × 传播时延（如果忽略处理时间）。

### 利用率

信道利用率 🔥 = 信道内有数据传输的时间占总时间的比率。利用率过高接近 1 → 排队时延急剧上升（拥塞崩溃）。网络应维持在适中利用率实现合理吞吐且低延迟。

### 丢包率

丢包率 🔥 = 丢失的分组数 / 发送的总分组数。超载网络的队列满载时到来的分组会被丢弃——这是 Internet 尽力而为服务（best-effort）不保证送达的直接表现。TCP 有对丢失的放重传反应。

## 常见题型

- **时延的计算（发送时延 vs 传播时延）**（绝对选择题高频）：给分组大小 L、发送速率 R、链路长度 D、传播速度 V——分别算出发送时延 L/R 和传播时延 D/V；指出哪个是更大成分。
- **分组交换与电路交换的时延对比**：给网络结构图，多条通信路径需要的跳数和转发器数量下两种交换的总时间——分析环节数——分组交换由于管线比电路交换产生更短的总时间。
- **网络性能指标的理解**：辨析「带宽」与「吞吐量」、发送与传播时延场景判断题。
