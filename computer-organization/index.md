# 计算机组成原理概述

> **考情提示**：本章属于概述性质，408 中一般 1–2 道选择题（2–4 分），核心考点是冯·诺依曼结构特点、CPU 执行时间公式、性能指标换算。属于「送分题」，性价比极高。

## 计算机发展历程

### 计算机硬件的发展

硬件史是四代计算机的迭代过程，考试只需要记住各代的标志性逻辑元件：

1. **第一代（1946–1957）**：电子管，体积庞大、功耗极高、可靠性差。ENIAC 是代表。
2. **第二代（1958–1964）**：晶体管，体积骤降、功耗大幅改善、开始使用高级语言。
3. **第三代（1965–1971）**：中小规模集成电路（SSI/MSI），把多个晶体管集成到一块硅片上，催生了操作系统。
4. **第四代（1972 至今）**：超大规模集成电路（VLSI），一块芯片上可集成数百万个晶体管，微处理器由此诞生。

**摩尔定律**：集成电路上可容纳的晶体管数量每 18 个月翻一番，性能也随之提升一倍。这是一种观察性规律而非物理定律。💀 考研中仅需知道摩尔定律的表述含义，不会深入考察其当前是否已逼近物理极限。

### 计算机软件的发展

软件经历了三级抽象提升的过程。**机器语言**是第一代——用二进制代码直接编程，可被硬件直接识别但人类完全无法阅读。**汇编语言** 🔥 用助记符代替二进制代码（如 MOV、ADD），通过汇编程序翻译成机器语言。**高级语言** 🔥（C、Java、Python）才是现代编程的起始点，通过编译器/解释器翻译为汇编或直接转换为机器码。

软件分为两大类：**系统软件**管理计算机自身（操作系统、编译程序、数据库管理系统）；**应用软件**解决特定领域问题（浏览器、办公套件、游戏）。

> 考研此处常出选择题：问「以下哪个是系统软件」——操作系统和编译程序是系统软件，浏览器和办公软件是应用软件。汇编程序本身是系统软件。

## 计算机系统层次结构

### 计算机系统的组成

一个完整的计算机系统由**硬件系统**和**软件系统**两大层次组成。硬件是物理基础，软件是逻辑层。两者之间通过**指令系统**（即机器语言）作为接口衔接。这种分层设计是计算机科学的精髓——每层只与相邻层通过明确定义的接口交互，不关心层的内部实现。

### 计算机硬件的基本组成

**冯·诺依曼结构** 🔥 是 408 必须烂熟的模型。核心思想只有一条：**程序和数据以同等地位存储在存储器中，按地址访问，按顺序逐条取出指令自动执行**。硬件由五大部件组成：

- **运算器**：完成算术和逻辑运算
- **控制器**：指挥整个系统的操作——根据指令产生控制信号
- **存储器**：存放程序和数据
- **输入设备**：将程序和数据送入计算机
- **输出设备**：将处理结果送出计算机

早期的计算机以**运算器为中心**（数据的输入输出都经过运算器），而现代计算机都以**存储器为中心** 🔥——简称「存储程序」体系。CPU 通过总线从存储器中读取指令和数据，运算结果再写回存储器。这就是为什么存储器性能对整个系统性能影响巨大（后续 Cache/虚拟存储章节的核心动机）。

### 计算机工作过程

计算机的工作本质上是反复执行取指→分析→执行的无限循环：

1. **取指令（Fetch）**：从 PC（程序计数器）指向的内存地址取出一条指令放入 IR（指令寄存器），PC 自增指向下一条指令。
2. **分析指令（Decode）**：译码器解析 IR 中的指令，产生相应的控制信号。
3. **执行指令（Execute）**：根据控制信号，数据通路完成相应的操作（运算、访存、跳转等）。

接着循环回到第一步。这三种节奏就是 CPU 指令周期的全部。考研中 PC 永远指向下一条待取指令（注意：取指后 PC 就加了），这是在画数据通路时最基本的动作顺序。

> 整个「程序是怎样运行的」本质上就是 PC → 取指令 → 译码 → 执行 → 更新 PC（或跳转改变 PC）→ 再取指令……无限循环。后面的 CPU 章节会详细拆解每一步的微操作和数据流动。

## 计算机性能指标

### 机器字长

**字长** 🔥 是 CPU 一次能处理的二进制数据位数，通常等于通用寄存器的宽度。64 位 CPU 的字长就是 64 位。字长越长，单次运算精度越高、直接寻址范围越大（地址空间 = 2^字长），但所需的硬件代价也越大。考研中最重要的是理解字长与数据通路宽度、寻址空间的关系——后面计算 Cache 容量和指令格式时全部用到。

### 数据通路带宽

数据通路带宽是 CPU 内部数据总线一次能并行传送的信息位数，不同于外总线宽度。外部数据总线的宽度影响一次能读取的内存信息位数，这直接影响 CPU 取指令速度。

### 主存容量

主存容量 = 存储单元个数 × 存储字长。例如 MAR（存储器地址寄存器）宽度为 10 位，则最大支持 2^10 = 1024 个存储单元；如果每个单元存放 16 位数据（即 MDR 宽度 16 位），总容量就是 1024 × 16 位 = 16,384 位 = 2KB。

### 运算速度

**CPU 时钟周期 = 1 / 主频**。例如主频 2GHz（= 2×10⁹ Hz），时钟周期就是 0.5ns。**CPI（Cycles Per Instruction）** 🔥 是 408 必考的参数：执行一条指令平均需要多少个时钟周期。这个数取决于指令集设计、实现效率和程序本身的指令组成。

**CPU 执行时间** 🔥 是计算的终极目标——也是 408 选择题最爱的计算式：

> **CPU 执行时间 = 指令总数 × CPI × 时钟周期**

或者等价于 = (指令总数 × CPI) / 主频。这个公式的三个因素分别受到不同的优化手段影响：
- **指令总数** → 编译器优化、算法效率
- **CPI** → 指令集设计、流水线技术
- **时钟周期** → 半导体工艺、微架构

**MIPS（Million Instructions Per Second）** 🔥 = 指令总数 / (执行时间 × 10⁶) = 主频 / (CPI × 10⁶)。MIPS 越高 CPU 速度越快？不全是——不同架构的指令干的事不一样多，MIPS 不能跨架构直接对比。

**MFLOPS（Million Floating-point Operations Per Second）** 🔥 专门衡量浮点运算性能，MFLOPS 和 GFLOPS（十亿次每秒）是现在衡量超级计算机排名的标准指标。

## 常见题型

- **冯·诺依曼结构的特点**：选择题常考「以下哪条不是冯·诺依曼机的特点」。要点是：程序和数据在存储器中地位相同、按地址访问存储器、指令顺序执行（除非跳转）。
- **CPU 执行时间的计算**（高频）：给出一段程序运行的指令条数、CPI、主频三个数，直接套公式 T = IC × CPI / f。注意单位换算（s/ms/ns 和 GHz/MHz/KHz）。
- **性能指标的理解和换算**：MIPS 值的计算、字长与地址空间的关系、存储器容量计算。注意区分 M（兆）在计算机中有时表示 10⁶，有时表示 2²⁰——408 一般按 2 的幂使用。
