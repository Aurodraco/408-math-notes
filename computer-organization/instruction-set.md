# 指令系统

> **考情提示**：指令系统是 CPU 设计的软件接口，在 408 中一般 1–2 道选择题（2–4 分）。扩展操作码的设计和寻址方式辨析是选择题高频考点，CISC/RISC 的区别几乎每年必出现在某个选项中。难度不高，性价比好。

## 指令格式

### 指令的基本格式

一条机器指令在计算机中存储和传递的形式是二进制代码，它包含两个部分：**操作码字段（OP）**和**地址码字段（A）**。操作码告诉 CPU 去做什么操作——加、减、传送、跳转等。地址码告诉 CPU 操作的对象在哪——源操作数在哪里、目标操作数放哪里、下一条指令在哪里。

操作码有两种设计思路：**定长操作码**和**扩展操作码**。定长操作码简单——所有指令的操作码占同样位数、位置固定，译码方便但不够灵活，浪费码点；扩展操作码在地址码不同长度的指令间复用操作码长度——当指令所需地址减少时，把没用的地址位划入操作码，从而容纳更多条指令。后文的扩展操作码设计会详讲。

### 地址码的个数

指令按地址码数量分类直接影响指令的二元特性：

- **三地址指令** 🔥：OP A1, A2, A3 —— '(A1) OP (A2) → A3'。最完整、最容易转化为高级语言的三元操作（如 `ADD R1, R2, R3`）。占用的二进制位数最多——每条指令需同时携带三个地址。
- **二地址指令** 🔥：OP A1, A2 —— '(A1) OP (A2) → A1' 或 'A2'。最常用的形式——现代 x86 和 ARM 的大多数算术指令均此格式。结果的地址码兼作源操作数存放位。
- **一地址指令** 🔥：OP A —— 隐式使用累加器（ACC）作为第二操作数和结果存放。比如将内存某单元的值加到 ACC，或者将 ACC 写入该单元。是早期计算机的标配，现在仅零散出现。
- **零地址指令** 🔥：只有操作码，没有明确的地址码——所有操作数都在**堆栈**上（栈顶作为源和目标）。典型例子是用堆栈结构计算器时的指令（如 PUSH、POP、ADD、MUL 等）。

> 408 选择题有可能让判断「这条指令是几地址指令」，通过指令的含义识别隐式参与的操作数来源与去向。

### 扩展操作码设计

扩展操作码 🔥 在实际 CPU 设计中被广泛使用：同一指令集中的不同指令可以有不同的操作码长度——地址多的指令占地址码多、操作码短（可容纳的指令种数少）；地址少的指令操作码可以变得更长（提供更多条不同的指令）。典型设计次序从最短操作码到最长：三地址指令的操作码最短 → 二地址长一些 → 单地址更长 → 零地址最长。

扩展操作码设计的核心问题是**指令条数的上限约束**：当某类指令的操作码没有被全部占用完时，剩余的编码数量可以「借」给更少地址的指令使用。考研常见的考题就是——给定指令总位数、各类型指令所需地址数及条数，让你算某类最多还能加几条。解题核心就是按三地址→二地址→一地址→零地址从上到下依次分配码点，看剩余量。

> 💀 实际 CPU 的扩展操作码设计远比考研作业题复杂——用到可变长操作码前缀和各种组合。408 只考基于固定指令字的简单扩展模型。

## 指令的寻址方式

### 指令寻址

指令寻址指的是找到下一条要执行指令的地址：

- **顺序寻址** 🔥：每执行完一条指令后，PC（程序计数器）自动加「1 条指令的长度」（即加字节数），指向下一条顺序待执行的指令。这是计算机正常运行 99% 以上的情况——程序顺序执行。
- **跳跃寻址** 🔥：当执行跳转指令（如 JMP、CALL、RET 等）时，PC 被强制定为跳转目标地址，打破顺序。跳跃寻址是程序分支和函数调用的硬件基础。

### 数据寻址

数据寻址方式 🔥 是指令决定「操作数的有效地址怎么算」的核心问题。以下按寻址范围从小到大排序说明：

- **隐含寻址**：操作数没有在指令中出现地址码——CPU 默认知道在某个固定寄存器中（如典型累加器 ACC）。如零地址堆栈指令也隐含在栈顶。指令字短，效率高但范围极其有限。💀 现在只有嵌入式 CPU 还使用纯隐含方式。
- **立即寻址** 🔥：指令地址码字段直接放操作数的值本身（不是地址）。执行速度最快——取完指令后数据已经在手边——但数的范围受指令字宽度的限制。广泛用于赋值立即常数（如 `MOV R0, #5`）。
- **直接寻址** 🔥：指令里给出的地址码就是操作数在内存的有效地址。取完指令后，用此地址访问一次内存就取得数据——总共两次访存（一次取指，一次取数）。比后面的间接寻址少一次访存——所存地址码中位数也直接限制了最大可直接访问的内存区域。
- **间接寻址**：指令的地址码不是有效地址，是指向「存放有效地址的内存字」的指针。一次二次访问内存才能取出数据。可访问更大的地址空间，但慢。💀 现代 CPU 极少使用纯一次或多重间接寻址——被视为过时，取而代之的是变址/基址寻址。考研了解概念即可。
- **寄存器寻址** 🔥：操作数在寄存器中，指令用寄存器编号指代。速度极快——不用访存。RISC 类型 CPU 几乎全部指令都用寄存器寻址。是现代计算机中数据寻址的主力。
- **寄存器间接寻址** 🔥：寄存器中存放操作数的内存地址。指令给出寄存器编号，先读寄存器得到地址，再通过地址访存取数。该方式既能灵活寻址大地址空间，又避免间接寻址多次访存的缺点——指针的底层实现。
- **相对寻址** 🔥：EA = (PC) + 偏移 A，偏移是指令地址码的常数字段。用于**转移指令**——跳转位置跟当前值多远。使得程序可以在内存中任意处载入执行（位置无关代码）——所有位置中的跳转偏移不需要固定绝对的地址值。
- **基址寻址** 🔥：EA = (基址寄存器) + 指令中的形式地址 A。基址寄存器保存一道程序或数据段在内存中的起始地址块。操作系统在装入程序时设置该寄存器——整段程序可在物理主存的不同位置加载而不改指令形式。配合分段。
- **变址寻址** 🔥：EA = (变址寄存器) + 指令中的形式地址 A。变址寄存器每次循环加 1 / 减 1 并连续通过一条指令就可在数组上遍历——连自编循环的拉址代码都不用写。这是**数组**访问的硬件基础。
- **堆栈寻址** 🔥：操作数在栈顶，指令操作隐式弹出/压入元素。用堆栈指针（SP）来管理栈顶部。是目前高级语言内部软实体的函数调用和表达式求值的重要方式。

### 基址寻址与变址寻址的对比

| 对比项 | 基址寻址 | 变址寻址 |
|--------|----------|----------|
| 目的 | 程序/数据段的重定位 | 数组、字符串等循环遍历 |
| 谁变谁不变 | 基址寄存器内容不变，A 变 | 变址寄存器内容变（每次循环自增），A 不变 |
| 典型应用 | 多道程序环境下的内存保护与对换 | `for` 循环中对 `a[i]` 的批量操作 |

> 408 几乎每两年考一次：「基址寻址和变址寻址哪个用于程序赋值？」答案基址是重定位、变址是数组循环。跨混淆就是错。

## CISC 和 RISC

### CISC（复杂指令集计算机）

CISC 🔥 设计哲学的核心理念是「用一条指令做更多的事」——指令数量多、长度不等、很多指令可以直接完成数个功能（如用一条指令实现字串复制）。便于写汇编程序，但硬件实现复杂——几乎必须微程序控制。典型的 CISC 代表是 x86，其特点包括：指令长度 1–15 字节不等、寻址方式极其丰富、大多数指令可以直接操作内存而无需先取到寄存器。

> 🔥 x86 至今仍是桌面和服务器市场的 CPU 统治指令集，但其内部早通过**微操作**将前端 CISC 转化为后端 RISC 风格。这是一种「外 CISC 内 RISC」的实现方式——吸取两者的长处。

### RISC（精简指令集计算机）

RISC 🔥 🔥 的核心设计原则是「每条指令做最少的事但做极快」——指令条数少、每指令长度固定、寻址方式简洁、大量通用寄存器配合 load/store 架构（只有 LD/ST 指令才访问内存，其余一切操作在寄存器间）。得益于指令极简，RISC 处理器**必须使用硬布线控制器**来实现更高的频率和更细粒度的流水线——效率远强于 CISC。代表是 ARM 🔥（移动端统治级）、MIPS ⚠️（教育市场）、RISC-V 🔥（新星开源指令集）。

> 🔥 ARM 和 RISC-V 的全面崛起说明历史潮流在往 RISC 方向走——今天的智能手机、IoT、云计算服务器几乎都系 RISC 或 RISC 内部实现。408 考 RISC 时最核心的应用性质就是「用流水线效率换取更高吞吐量」。

### CISC 与 RISC 的对比表

| 对比维度 | CISC | RISC |
|----------|------|------|
| 指令数目 | 多（数百） | 少（数十至百余条）|
| 指令长度 | 不等 | 固定 |
| 寻址方式 | 多种多样 | 少且简单 |
| 访存方式 | 大多数指令可访存 | 仅 Load/Store 指令访存 |
| 控制器 | 微程序控制为主 | 硬布线（组合逻辑）控制为主 |
| 流水线 | 较难高效流水线 | 天生易于流水线实现 |
| 编译复杂 | 编译简单，一条指令完成多项任务 | 编译复杂，需要更多的优化支援 |
| 代表 | x86 | ARM, MIPS, RISC-V |

> 408 这些年明显加大对 RISC 的考问——特别是 RISC 流水线优势和对指令负荷的依赖。

## 常见题型

- **扩展操作码设计**：给定指令长度和各种地址类指令的数量要求，计算某类还能容纳多少条。方法是从地址最长类往下依次分配操作码种子，看剩余的码点数量。
- **各种寻址方式的有效地址计算**：给指令的地址码字段值、PC 值、相关寄存器的值，判断每种寻址方式下有效地址 EA 是多少以及需要几次访存。
- **CISC/RISC 特点区分**：最常见是对比选择题——「以下哪项是 RISC 的特点」「哪个是 CISC 的特点」。记住核心词：RISC = 固定长度 + Load/Store + 硬布线 + 良好流水线；CISC = 不等长度 + 内存操作数 + 微程序控制。
