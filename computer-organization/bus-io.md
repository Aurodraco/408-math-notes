# 总线与输入输出系统

> **考情提示**：总线与 I/O 是组成原理最后一章，一般在 408 中出 1 – 2 道选择题（2 – 4 分）。重点：总线带宽计算、三种 I/O 方式对比、中断处理流程。难度中等，偏记忆型。

## 总线

### 总线的基本概念

总线 🔥 是连接多个部件的公共信息传输线——计算机的神经系统。各个部件都通过同一组线分时地传送信息。总线上任何时刻只能有一个部件发送信息，但可以有多个部件同时接收。总线设计需要解决的核心问题：谁可以在什么时候使用总线。这由总线仲裁和定时机制解决。

### 总线的分类

按连接范围分三类：
- **片内总线**：CPU 芯片内部连接运算器与控制器等部件的数据通路——对程序员不可见。如 ALU 到寄存器之间的连线。
- **系统总线** 🔥：连接 CPU、主存和 I/O 接口的那一组总线。可分为数据总线、地址总线和控制总线三组。
- **通信总线**：用于计算机系统之间或计算机与外部设备间的远距通信（USB、PCIe 等）。

按系统结构分：
- **单总线结构**：CPU、主存、I/O 全部挂在同一总线上——结构简单、易于扩展，但 I/O 和 CPU 无法同时使用总线，效率低。
- **双总线架构**：CPU 与主存通过专有的**存储总线**通信，I/O 再通过另一条 I/O 总线与主存交互（DMA）。比单总线高效。
- **三总线结构**：再加一条局部总线——存储总线、I/O 总线和局部高速总线同时存在。其实现是最广泛的现代计算机系统形式。

### 总线的性能指标

- **总线宽度** 🔥：通常指数据线的数量（一次能并行传输多少位）。常见宽度 32、64、128……。
- **总线频率** 🔥：总线的时钟频率——每秒钟传输周期的次数。
- **总线带宽** 🔥 🔥 = 总线宽度 × 总线频率 × 每周期传输次数。例如 64 位数据总线、100MHz 时钟、每题传输 1 次 → 带宽 = 64b × 100×10⁶ Hz = 6400 Mbps = 800 MB/s。

> 408 几乎每年一道总线带宽计算选择题——一个乘公式的秒杀题。注意最终结果按照题目要求的单位（B/s 或 b/s 或 Mb/s）转换。

### 总线仲裁

解决多个主设备同时请求使用总线的冲突——究竟谁拥有总线使用权。仲裁方式分两大类：

**集中仲裁** 🔥：由一个中央仲裁器统一裁决：
- **链式查询**：设备按优先后优先排序连成一条链，仲裁器发出准许信号后沿链传递。第一个需要总线并且信号到位时截获并占用总线。实现简单，但优先级固定且链断则后面全失联。💀 已经较少采用。
- **计数器定时**：仲裁器中的计数器循环计数，当计数值与某设备编号匹配且该设备有请求时获取总线。各设备优先级相等。比链式慢，但抗断链。
- **独立请求** 🔥：每个主设备有自己单独的请求和应答线。仲裁器通过复杂的排队及优先级逻辑在多个请求者间仲裁。最快最灵活，是现代多数系统采用的方式。

**分布仲裁** ⚠️：不使用中央仲裁器，各设备自带仲裁逻辑并通过自判决决定优先级。适用于多处理器系统。

### 总线操作和定时

**同步定时** 🔥：所有收到控制信号的主/从设备都在总线时钟的固定节拍上动作。双方不需要应答——跟着时钟周期即可。优点是高速（完全流水化），缺点是对介质长度/负载差异的宽容度差，必须满足最慢设备的要求。

**异步定时** 🔥：不依赖统一时钟，而是通过请求-应答**握手**收发完成每一次传送。分为三种握手：
- **不互锁**：主方发出请求后经固定时间向接收设备写入数据并随之撤销请求——不等对方回应。
- **半互锁**：主方发出请求后收到应答后再撤销请求。一方等待对方确认但另一方不需要反确认。
- **全互锁** 🔥：主方发请求、从方接受到数据给应答、主方收到应答后撤消请求、从方见请求取消后也撤销应答。最稳但慢。用于异步可靠通信——多数异步总线采用全互锁方式。

### 总线标准

| 标准 | 类型 | 带宽 | 当前状态 |
|------|------|------|:--:|
| **ISA（工业标准总线）** | 并行系统总线 | 8/16 位，几十 MB/s | 💀 已完全过时 |
| **PCI** | 并行系统总线 | 33/66 MHz，133–533 MB/s | ⚠️ 已较少使用，被 PCIe 替代 |
| **PCI Express（PCIe）** | 高速串行点对点 | 几 GB/s 至数十 GB/s | 🔥🔥 现代计算机标配 |
| **USB** | 外设串行总线 | 逐年升级 12 Mbps–40 Gbps | 🔥🔥 通用外设连接 |
| **SATA** | 存储设备总线 | 1.5–6 Gbps | ⚠️ SSD 正被 NVMe 替代 |
| **NVMe** | 高速 SSD 总线（基于 PCIe） | 数十 GB/s | 🔥🔥 当前 SSD 主力 |

> 408 考总线标准只涉及基本辨认——知道 PCIe 是当前主流串行总线、USB 是外设串行标准即可。不考带宽数值。

## I/O 接口

### I/O 接口的功能

I/O 接口 🔥 位于系统总线与外部设备之间，完成数据传送中的缓冲和信号转换。主要功能：
- **数据缓冲**：CPU 和外设速度不同步，接口内建小寄存器缓冲区暂存数据。
- **信号转换**：外设的电信号可能与计算机系统总线不全兼容——接口完成电压电平、并行/串行格式等的转换。
- **地址译码**：当 CPU 的设备地址信号与接口配置的地址匹配时，接口被选中并连通内部的控制与数据寄存器。
- **通信控制**：向外设发送控制命令和接收状态信息。

### I/O 接口的组成

I/O 接口从 CPU 的角度看就是一组寄存器：
- **数据端口** 🔥：存放 CPU 要送给外设的数据或刚从外设接收到的数据。
- **状态端口**：描述外设的当前状态（忙/闲/就绪/错误等）。CPU 通过读这个端口判断是否可以开始新的传送。
- **控制端口**：CPU 通过写这个端口发出控制命令给外设（如启动/重置/工作方式设定等）。

所有端口都有一个唯一的 I/O 地址，CPU 通过地址选择某一设备的某个端口进行读写操作。

### I/O 端口及其编址

两种编址方式决定 CPU 如何访问 I/O 接口的端口：

**统一编址（存储器映射 I/O）** 🔥：把 I/O 端口视为主存地址空间的一部分——通过常规的 LOAD/STORE 指令就可以访问 I/O 端口。优点是不需要额外的 I/O 指令、所有访存指令也可用于外设操作。缺点是挤占主存地址空间（减少了有效可用的内存地址上限）。ARM 架构采用统一编址。

**独立编址** 🔥：I/O 端口有自己独立的地址空间，通过专用的 I/O 指令（如 x86 的 IN/OUT）访问。内存地址空间全留给主存。x86 采用独立编址。

> 408 选择题常问「哪种编址需要专门的 I/O 指令」——独立编址。两种编址各自的缺点：统一编址挤占内存地址空间；独立编址需额外 I/O 指令。

## I/O 方式

### 程序查询方式

程序查询方式 🔥（轮询）是最简单的 I/O 控制：CPU 发送一条命令给外设后，不断循环查询外设状态寄存器——若外设尚未就绪就原地「自旋」直到状态变为就绪，然后执行数据传送。过程全部由 CPU 操作，期间 CPU 不能处理任何其他任务——CPU 和外设是串行工作的。优点是最简单的实现，额外硬件成本为零（仅需状态端口）；缺点是 CPU 全部期间耗费在查询之中使机效率极低。

> 💀 在实际通用计算机中纯粹循环轮询极少使用（除非预期极短的延迟如少量微秒），嵌入式系统在简单单任务时仍可能用轮询。

### 程序中断方式

程序中断方式 🔥 🔥 是 I/O 控制的质的跃升：CPU 发出 I/O 命令后不再等待，继续执行当前程序。当外设完成数据准备后，发出**中断请求**到 CPU。CPU 在执行完当前指令后响应中断，中断处理程序完成少量快速的数据传送（通常一个数据块中的一个字）后返回原程序继续执行。全程 CPU 与外设实现了并行。

中断程序需处理的内容：读取外设数据寄存器->存入内存缓冲区->更新指针->准备下次中断；遇到完整一次缓冲区满后才唤醒阻塞的程序。

**中断屏蔽** 🔥：CPU 可设置中断屏蔽位（PSW 中的 IF），清除则所有的可屏蔽中断全部被屏蔽。还可以为每个 I/O 控制器单独设置屏蔽控制寄存器实现精细化的优先级屏蔽。

**中断响应周期** 🔥：这是一小段纯硬件流程。当 CPU 在两条指令之间被中断线拉高且满足允许条件时——硬件自动：① 关中断（保存可屏蔽到后续中断）② 保存当前断点 PC 和 PSW 压入系统堆栈 ③ 从中断向量表根据中断类型 n 获取对应的中断服务程序入口地址装入 PC → 从而开始执行中断处理程序。

**单重中断**和**多重中断**的区别在于是否允许更高优先级的中断嵌套打断当前正在执行的中断服务程序。单重关中断期间不嵌套；多重允许嵌套——对应高档的优先级系统。

### DMA 方式

DMA（Direct Memory Access） 🔥 🔥 是高速大批量数据传送的王牌。专门有 DMA 控制器芯片来负责在 I/O 设备和主存之间直接进行数据传送——期间 CPU 不再参与。DMA 与 CPU 争主存总线的使用权。

DMA 控制器的核心配置量包括：内存起始地址、传送字数、传送方向（读/写）等。CPU 在给 DMA 控制器配置这些信息后，即可忘掉这次传送——在外设和主存之间批量传送由 DMA 独立完成。结束后 DMA 控制器发出一次中断通知 CPU。

DMA 传送的三种方式：
- **停止 CPU 访问主存（成组传送）**：DMA 全部占用总线传输整块数据——期间 CPU 暂停执行（保持保持但不动内存）。适合整块数据大规模一次搬完。
- **周期挪用（Cycle Stealing）** 🔥：DMA 在 CPU 不使用总线的时候偷偷占用一两个总线周期传送一个字，不中断主程序运行。是 DMA 最常用的方式。
- **CPU 与 DMA 交替访问**：将时钟周期分成两个时隙分别给 CPU 和 DMA 轮流使用——硬件设计最复杂。

DMA 的处理过程分三步：预处理（CPU 写命令和参数到 DMA 控制器）→ 数据传送（DMA 控制器独立传送整批数据）→ 后处理（完成中断时 CPU 检查传送状态）。

| 对比项 | 程序查询 | 程序中断 | DMA |
|--------|----------|----------|-----|
| 数据传送驱动 | CPU 主动循环 | 外设通过中断请求 | DMA 控制器独立 |
| CPU 与外设并行 | 否 | 是 | 是 |
| 适用速率 | 极低速 | 中低速 | 高速块传送 |
| 额外硬件 | 无 | 需中断受理电路 | 需 DMA 控制器 |

### 通道方式

通道 🔥 是一种比 DMA 更高级的 I/O 处理部件——相当于一个专门的 I/O 协处理器，专管对外设的控制和对主存的传送。CPU 用通道指令编制通道程序，然后启动通道让通道独立执行完整个通道程序，全面控制多个设备进行复杂的 I/O 序列。

通道有三类：
- **字节多路通道** 🔥：交替为多个低速设备服务——按字节轮流传输。适合键盘、打印机等低速设备。
- **数组选择通道**：一次性为一台高速设备完成整个数据块传送——期间通道独占该设备。适合磁盘。
- **数组多路通道**：可同时为多台高速设备服务——以数组为单位轮流在多磁盘间切换传输。是现代大型机的主流 I/O 通道结构。

> 💀 DMA 和通道的内容在 408 出现的几率较小，考察为主观「大型机通道与 DMA 的区别」对比题。绝大部分考生了解通道的基本概念和三种类型的适用场景足够。

## 常见题型

- **总线带宽计算**：给线宽、频率、每周期传几次——乘法算出带宽。注意转换为题目指定的单位（MB/s 或 Gb/s）。
- **程序查询/中断/DMA 三种方式的对比**：常见选择题。选特点：哪个 CPU 与外设可并行？哪个由 CPU 不停轮询？哪个由专门硬件控制器不需 CPU 逐字参与？
- **中断处理流程图的绘制**：从头画接口异常请求到返回的全步骤图，包括关中断、向量表查找、处理程序、现场保存恢复等。要配合 PSW 动作。
- **DMA 传送过程中的总线使用权问题**：周期挪用方式下的 DMA 和 CPU 怎样交替使用系统总线——择时冲突分析（考研偶尔出大题）。
