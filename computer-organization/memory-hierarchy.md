# 存储层次

> **考情提示**：本章是计算机组成原理的核心章节之一，Cache 地址映射和虚拟存储器地址转换是历年大题的最高频题材（≈10 分），另外主存芯片扩展 + DRAM 刷新也常出现在选择题中（2–4 分）。本章学好后，操作系统内存管理章节的理解难度也会大幅降低。

## 存储系统概述

### 存储器的分类

存储器可以从三个维度分类：

**按存储介质**：**半导体存储器**（内存、Cache）🔥 速度最快；**磁表面存储器**（磁盘）⚠️ 现在逐渐被 SSD 替代但仍在大规模数据中心使用；**光存储器**（光碟）💀 已基本过时，考研仅了解。

**按存取方式**：**随机存取存储器（RAM）** 🔥 可按任意地址直接访问，存取时间与位置无关——主存是 RAM；**只读存储器（ROM）** 出厂后只能读出不能随便写；**顺序存取存储器（SAM）** 必须顺序扫描找到目标——磁带属于此类 💀。

**按在计算机中的作用**：主存（存放正在运行的程序和数据）、辅助（存放长期保存的信息）、高速缓冲存储器（Cache）。

### 存储器的性能指标

- **存储容量** = 存储单元数 × 存储字长（位或字节）
- **存取时间**：从发出读写命令到完成数据存取的时间
- **存储周期** 🔥：连续两次对存储器进行访问的最小时间间隔——通常比存取时间略长，因为存储单元需要恢复时间。这是存储器的速度关键参数。
- **存储器带宽** 🔥：单位时间内能从存储器读写的总信息量，单位 b/s 或 B/s。带宽 = 总线宽度 × 总线频率 × 每周期传输次数（对 DDR 有双传输）。

> 408 选择题常考「存储周期和存取时间的区别」——存取时间是完成一次存取的最小时间，存储周期是整个存储器能接纳第二次请求的最小间隔。

### 存储系统的层次结构

现代计算机的存储器是分层体系：**寄存器**（CPU 内部，ns 级）→ **Cache**（SRAM，ns 级）→ **主存**（DRAM，十 ns 级）→ **辅存**（SSD/磁盘，μs ms 级）。越往上层速度越快、容量越小、价格越贵。层次结构的设计原则是**上层是下层的缓存**——上层存了下层的一部分副本，让 CPU 感觉整体存取速度接近上层。

层次化的实质是：把当前最频繁使用的代码和数据调度到最快的存储器中，不常用的留在下一级。这个调度的自动化和透明性是硬件和操作系统共同完成的。

### 程序的局部性原理

程序的局部性原理 🔥 是整个存储层次体系存在和工作的理论基础——没有局部性，Cache 和虚拟存储器毫无意义。分两种：

- **时间局部性**：刚被访问过的数据可能在不久的将来再次被访问。典型的例子是循环变量和频繁调用的函数。
- **空间局部性**：刚被访问过的单元附近的单元很可能随后被访问。典型的例子是顺序执行的程序和数组元素的连续访问。

> 408 选择题常见问题是给一段 C 语言代码，判断里面哪些引用是时间局部性、哪些是空间局部性。关键标准：循环体中的变量（特别是累加变量）是时间局部性；数组的相邻元素是空间局部性。

## 主存储器

### SRAM 和 DRAM

**SRAM（静态 RAM）** 🔥：用**双稳态触发器**存储信息——一个位存储元是含 6 个晶体管的触发器电路。只要有电就保持数据不变，不需要刷新电路。速度快（存取时间几纳秒）、功耗较高、集成度较低、价格昂贵。用作**Cache**。

**DRAM（动态 RAM）** 🔥：用**电容**存储信息——电容充满电表示 1，无电荷表示 0。电容会漏电，必须在信息消失前重新充电，这叫做**刷新**。DRAM 的优点是集成度高、成本低、功耗少——缺点是速度比 SRAM 慢，且需要定期刷新。用作**主存**。

**DRAM 的刷新** 🔥 是所有刷新方法选择题的高频点：

| 刷新方式 | 原理 | 特点 |
|----------|------|------|
| **集中刷新** | 在一段集中时间内对全部行逐一刷新 | 实现简单，但有「死区」——CPU 在这一整段时间不能访存 |
| **分散刷新** | 将各行的刷新散开到各存取周期末 | 无死区，但把存储周期拉长 |
| **异步刷新** | 在规定的刷新周期内将各行刷新均摊到各时间片 | 折中方案，**为大多数现代系统所用** 🔥 |

> 408 选择题最爱问刷新时间分配的判断：给出最大刷新间隔（如 2ms）、行数（如 4096 行），问每行刷新间隔是多少——分散刷新为 2ms/4096 ≈ 488ns 一次，这意味着存取周期被大幅拉长。

**行列地址复用**：DRAM 地址信号分两次传送——先把行地址（RAS）放入再送列地址（CAS），从而地址引脚只需要芯片地址线的一半。这就是 DRAM 容量远大于 SRAM 但不影响引脚数目的关键。

### 只读存储器（ROM）

ROM 🔥 是只读存储器——正常运行时只能读出不能随便写入。整趟存储体系按从最完全非易失到可擦写的发展过程分：

- **MROM**：掩模只读存储器——生产时已固化程序，不能改。💀 过时，考研知道是 ROM 的一类即可。
- **PROM**：可编程只读存储器，用户可自行写入一次——不可擦除。
- **EPROM**：紫外线可擦除可编程 ROM，通过专用的紫外线窗口擦拭后可重新写入。⚠️ 已被替代，考了解。
- **EEPROM**：电可擦除可编程 ROM，以电信号直接擦除和重写数据——擦写次数有限。🔥 曾用于嵌入式固件存储。
- **Flash Memory** 🔥：基于 EEPROM 技术的高密度存储设备，广泛用于固态硬盘、U 盘、SD 卡和 BIOS 存储。支持块擦除，擦写次数可达数百万次。

> BIOS（基本输入输出系统）传统上使用 ROM/Flash Memory 存储——其内容在电脑出厂时被烧录，而每次开机 CPU 从 BIOS 中执行第一条指令。

### 主存储器的构成

主存芯片的容量扩展是 408 年年必考的一项纯计算技能 🔥。通用公式：（目标容量）÷（芯片容量）= 需要的芯片数量。但需区分三种扩展方式：

- **位扩展** 🔥：增加数据线宽度——用多个芯片组成一组，共同提供完整的字长。如用 4K×4 位的芯片构建 4K×8 位的系统，需要 2 片一组并行给数据。
- **字扩展** 🔥：增加存储单元数——通过多组的地址线扩展可寻址空间。如用 4K×8 位的芯片构建 8K×8 位的系统，需要 2 组 4K 个字的芯片通过地址线高位经过片选信号选择是哪组。
- **字位同时扩展**：地址和数据都需要扩展——最常见的实际场景。例如用 1K×4 位的芯片构建一个 4K×8 位的系统，容量需要 4 组字向 × 2 片位向 = 8 片芯片。

**片选信号（CS）** 🔥 的生成较位扩展中扮演关键角色。通常使用主存地址的高若干位（超出单芯片可寻址范围的位）经过译码器产生芯片选择信号：**线选法**——直接使用高位地址线分别选芯片（简单但可能出现空地址）；**译码片选法**——通过专门译码器电路将 n 位地址译为 2ⁿ 个片选信号（更为标准）。

## 高速缓冲存储器（Cache）

### Cache 的基本工作原理

Cache 🔥 是放在 CPU 与主存之间的小容量高速存储器——全是 SRAM。CPU 要访问内存时先把地址发给 Cache，如果命中（数据在 Cache 中），直接返回，无需访问缓慢的 DRAM 主存；如果不命中，Cache 从主存取来整个块（以块为单位，通常 64 字节）存放，同时把 CPU 需要的数据返回。Cache 的设计有两个核心问题：**地址映射**（主存块放到 Cache 的哪里）和 **写策略**（写操作如何同时更新 Cache 和主存）。

### Cache 与主存的映射方式

**直接映射** 🔥：每个主存块只有一个固定可放的 Cache 位置——Cache 行号 = 主存块号 mod Cache 行数。优点是硬件极简单，通路的逻辑电路只需少量比较器。缺点是冲突多，主存中多个映射到同一 Cache 行的块会相互驱逐，**命中率较低**。CPU 地址被分解为三个字段：标记（主存块号的高位部分） | Cache 行索引 | 块内偏移。

**全相联映射** 🔥：主存中的任何块可以放入 Cache 的任何行。冲突概率最低，**命中率最高**，但硬件实现最复杂——需要对整个 Cache 的所有标记做并排比较（全相连比较器），成本随着 Cache 增大而暴涨。适用于小容量如 TLB。

**组相联映射** 🔥 🔥：现代 CPU 实际使用的折中方案。把整个 Cache 分成若干组，每组内含多个行（常见 4 路、8 路、16 路组相联）。主存块固定映射到唯一一个组内，但可在该组的所有行中自由选择位置。组内搜索代价等于组内行数（4 路 = 4 个比较器），远小于全相联。组相联的地址结构为：标记 | 组索引 | 块内偏移。

> 408 大题必考：给一个内存地址（十六进制），要求写出标记、组索引/行索引、偏移各自占多少位，并识别出在哪种映射方式下，该访问是否 Cache 命中。掌握了地址结构三字段的计算就掌握了这题的核心。

### 替换算法

当新块需要装入 Cache 时，如果指定的 Cache 位置已满，必须把某块数据「踢出」，腾出空间。替换算法只对**全相联与组相联**有意义（直接映射的块位置固定，无选择余地）。

- **随机替换（RAND）**：随机选一个块替换。实现最简单，命中率较低。
- **先进先出（FIFO）**：踢出最早装入的块。实现不复杂，但不适合局部性良好的应用场景。
- **近期最少使用（LRU）** 🔥：替换最久未被访问的块——这是目前最常用的方法，命中率和实现成本的平衡点最好。对于 4 路组相联，LRU 可以只用少量状态位实现。
- **最不经常使用（LFU）** ⚠️：替换访问次数最少的块。需要计数器累积访问次数，实现代价高于 LRU，而且有「旧数据长期不过期」的风险。很少被实际 CPU 采用。

> 408 选择题经常问：某个访问序列在给定替换算法和映射方式下的命中次数——这需要一步步画 Cache 状态。LRU 和 FIFO 是最常考的使用场景。

### 写策略

**写命中时** 🔥：CPU 写入了 Cache 中已有的一个块——怎么同时更新主存？

- **全写法（Write Through）**：写 Cache 的同时立即写入主存。保证 Cache 与主存始终一致，但写主存速度慢导致频繁停顿。一般加一个**写缓冲**来缓解。
- **写回法（Write Back）**：只写 Cache，在被替换出去时一次性整块写回主存。实现上需要为每个 Cache 行新增一个「脏位」记录是否有过写的发生。减少了主存写入操作，现代 CPU 大多使用写回法 🔥。

**写不命中时** 🔥：CPU 写入的主存块当前不在 Cache 中——要先把该块装入还是直接写主存？

- **写分配法**：先把该块从主存调入 Cache，再按写命中规则处理（通常配合写回法）。
- **非写分配法**：绕过 Cache 直接写入主存，不做 Cache 装入（通常配合全写法）。

> 408 选择题重要规律：写回法一般搭配写分配法，全写法一般搭配非写分配法。考试出过「哪种组合不符合常理」。

### Cache 性能分析

Cache 性能的核心公式 🔥：

> **平均访问时间 = 命中率 × Cache 存取时间 + (1 − 命中率) × 主存存取时间**

或等价地用下式计算（访 Cache + 缺失惩罚）：

> **平均访问时间 = Cache 访问时间 + (1 − 命中率) × 缺失损失**

408 选择/填空常要求指定 Cache 容量提高后，命中率上升能带来多少平均访问时间的改善、或者读出一段文件的平均存取时间。

Cache 容量越大，命中率越高，但成本更高、访问延迟也略微上升（更大的相联搜索回路需要更长的译码时间）。现代 CPU 通常通过**多级 Cache** 来平衡：L1 Cache 小而快（仅 32–64KB），L2 Cache 中容量（256KB–1MB），L3 Cache 大而全（数 MB–数十 MB）。考研了解多级 Cache 的概念即可，不算复合内容。

## 虚拟存储器

### 基本概念

虚拟存储器 🔥 🔥 是操作系统提供的抽象，让程序员感觉拥有连续统一的大内存，但实际上物理主存只装载活跃部分。程序使用**虚拟地址（逻辑地址）**，CPU 发出虚拟地址后，由硬件（MMU）配合操作系统中的**页表**转换为实际的**物理地址**。

**页表**：是操作系统为每个进程独立维护的数据结构，记录每一虚拟页是否在物理页中、若在以哪一物理页框、以及访问权限等。页表存储在物理主存中（大得太大了），所以每次地址转换都需访问页表所在的内存，从而多一次访存。

**快表（TLB / Translation Lookaside Buffer）** 🔥：专门为页表做高速缓存的硬件——存储最近用过的页表项。现代 CPU 中，TLB 在访问页表前先被查询，若命中直接获得物理页框号，省去一次主存访问。TLB 一般为全相联或高路组相联映射，容量极小但至关重要（几十到几百项）。

### 页式虚拟存储器

整个物理主存和虚拟页都划分成**等大固定大小的页**（通常 4KB 🔥）。页式管理下，物理页框可以装入任意的虚拟页——这是页表存在的原因。页表的每一项包含：有效位、物理页框号、访问控制位（读/写/执行）、脏位（记录页被写入过）、最近访问位等。

**页表基址寄存器**（如 x86 的 CR3）：CPU 的一个专用寄存器，存放当前运行进程的页表在物理主存中的首地址。上下文切换时操作系统只需更新这个寄存器即可切换进程的虚拟地址空间。

**多级页表** 🔥：在现代 64 位系统中，虚拟地址空间极大（2^48–2^64），单一线性页表体积无法承受。多级页表只建那些真正在使用的中间目录层页表，不使用的区域不用分配。x86-64 使用四级页表（PGD→PUD→PMD→PTE），每多一级都增加一次内存访问。这是 TLB 如此重要的原因——运气好 TLB 命中即跳过所有多级页表访问。

### 段式虚拟存储器

段式管理 ⚠️ 把程序的逻辑片（代码段、数据段、栈段等）作为按段大小不固定的小分配单元。地址包含段号 + 段内偏移，转换时需查段表。段式在提供保护方面自然适合（每段可独立设权限），但内存分配不固定而产生外部碎片。💀 现代操作系统几乎不再单独使用纯粹的段式，x86 是分段+分页的混合。

### 段页式虚拟存储器

段页式结合了段式的易编成保护的分段逻辑和页式的无碎片内存利用率——程序先按段分、段内再按页分。虚拟地址 = 段号 + 段内页码 + 页内偏移。每进程一组段表指向一组页表。考研只要知道地址转换需要两次查表——先查段表得页表基址、再查页表得物理页框。

### 虚拟存储器与 Cache 的比较

| 对比维度 | Cache | 虚拟存储器 |
|----------|-------|------------|
| 目的 | 遮蔽 CPU 与主存的速度差距 | 扩大主存的逻辑容量 |
| 管理方 | 硬件自动 | 操作系统主导 + 硬件辅助 |
| 数据块单位 | 块（Block）/行（Line） | 页（Page） |
| 未命中损失 | 访问主存，几十ns | 磁盘访问（缺页中断），几百万 ns |
| 实现映射 | 直接/组相联 | 操作系统页表映射 |
| 等效访问公式 | 命中率 × Tcache + 缺失率 × Tmem | （TLB 命中率 × Tmem） + 缺页率 × Tdisk |

> 408 常考两者的相同点和不同点对比，以及两者的配合工作（TLB + Cache + 主存 的联合访问流程）。

## 常见题型

- **Cache 地址映射的命中分析**（大题必考）：给主存地址 16 进制、Cache 容量、块大小和映射方式（经常是组相联），画出地址字段分割，推演在一个给定的访问序列下命中数和最终的 Cache 状态。
- **主存芯片扩展**：给芯片规格和要构建的存储器规格，计算所需芯片数，画片选译码电路，写出每个芯片的地址范围。
- **虚拟存储器地址转换（TLB + Cache + 页表 联合工作）** 🔥（整合综合分析题）：CPU 发出虚拟地址 → 切出 VPN + 偏移 → TLB 命中直接得 PPN + 偏移 → 物理地址上用 Cache 搜索（如果 Cache 命中直接得到数据）→ 如果 TLB 不命中需访问页表（可能再引发缺页中断）。每一步的可能结果都需被分析——这是 408 大题中最复杂的综合流程。
- **DRAM 刷新周期计算**：给出行数和最大刷新间隔，计算各种刷新策略下的时间参数和 CPU 死区比例。
