# 中央处理器

> **考情提示**：CPU 是计算机组成原理的终章和集大成章——数据通路分析、流水线性能计算、微程序控制器是大题三大王牌。本章几乎每年必出一道大题（≈10 – 15 分），是决定 408 得分高低的分水岭。

## CPU 的功能和基本结构

### CPU 的功能

CPU（中央处理器）是计算机的指挥与运算中心，负责执行程序中的所有指令。它必须具备以下五大功能：

- **指令控制** 🔥：确保程序按正确的顺序执行。通过 PC（程序计数器）自动取下一指令，遇到跳转则更新 PC 以改变流向。
- **操作控制** 🔥：分析指令的操作码，产生所需的各种微操作控制信号，指挥各部件完成动作。
- **时间控制** 🔥：为每条指令的每个操作周期（一个或多个时钟周期）分配精确的时间序列——即所谓的「时序」。控制单元根据节拍产生各个微操作信号。
- **数据加工** 🔥：执行指令的具体数据操作——算术运算（加、减、乘、除）和逻辑运算（与、或、非、移位）。
- **中断处理** 🔥 🔥：对突发事件（如 I/O 完成/缺页异常/硬件错误）作出反应。需要及时保存当前指令的现场，转去执行中断处理程序，处理完毕再恢复。

### CPU 的基本结构

CPU 内含两大部件——**运算器**和**控制器**，两者通过 CPU 内部总线连接。

**运算器** 🔥 的核心是 ALU（算术逻辑单元），加上一组通用寄存器、状态寄存器和移位器。ALU 接收两个输入操作数，根据控制信号选择执行何种运算，输出结果和标志信号（零/符号/溢出/进位）。

**控制器** 🔥 是 CPU 的「大管家」。它从 IR（指令寄存器）中获得正在执行的指令，经由译码器分析后，在节拍发生器的配合下，依序发出微操作控制信号操控整个数据通路。控制器的两种实现方式——**硬布线**和**微程序控制**——是后文的重点。

**专用寄存器** 🔥（必须烂熟于心的 6 个大寄存器）：
- **PC（程序计数器）**：永远指向下一条待取指令的内存地址。取指后自增（+指令长度），跳转时被目标地址覆盖。
- **IR（指令寄存器）**：保存当前正在执行的那条指令。指令取出后先放入 IR，控制器读出其 OP 和地址码。
- **MAR（存储器地址寄存器）**：存放即将访问的内存的地址——每种访存操作前，CPU 先把地址打入 MAR，再启动读写。
- **MDR（存储器数据寄存器）**：存放刚从内存读出的数据或即将写入内存的数据，是 CPU 与主存之间的数据缓冲区。
- **PSW（程序状态字/标志寄存器）**：记录上条指令执行结果的特征位——如零标志 ZF、符号标志 SF、溢出标志 OF、进位/借位标志 CF。条件跳转指令就是根据 PSW 的位值决定是否跳转。

> 408 喜欢向新生查问这些寄存器中哪些是程序员可见的（PC、PSW、通用寄存器），哪些是系统隐含的（MAR、MDR 在大多数架构中对程序员不可见）。必须分清！

## 指令执行过程

### 指令周期

CPU 运行的基本节律就是**指令周期** 🔥——处理一条指令的完整时间过程。经典模型划分为四个子周期：

1. **取指周期（Fetch）**：PC→MAR→读命令→MDR→IR，PC 自增。CPU 从内存取来指令。
2. **间址周期（Indirect）**：（如果指令使用间接寻址）使用指令地址码从内存取出有效地址。现代的简单指令大多不需要间址周期，但早期计算机几乎所有指令都要间址。
3. **执行周期（Execute）**：完成指令的核心动作——运算、传送、访存、跳转等。
4. **中断周期（Interrupt）**：指令结束后，CPU 检查有没有外部中断请求——如果有，保存 PC 和 PSW 等现场，装入中断处理程序入口地址，转入中断处理。

这四种周期信号经常在数据通路图中用触发脉冲表示周期跳转。不同的指令类型会有不同的周期调用组合——比如 ADD 指令可能只有取指周期和执行周期两个阶段，而间址寻址又多一个间址周期。

### 数据通路

数据通路 🔥 🔥 是信息在 CPU 各寄存器、运算器和总线等部件之间流动的**具体路线**。

- **单总线结构**：所有寄存器共享一条内部总线。结构最简，但每个时钟周期只能在总线上传一个数据，ALU 操作需要多个步骤完成。
- **双总线结构**：两条内部总线——可以同一时钟在两总线上并行传送两个操作数，大幅度提高运算速度。是 RISC CPU 的主流设计 🔥。
- **三总线结构**：再增加第三条总线——同时提供两个源数据和写入目标寄存器的通路。速度最快，但硬件代价也最高。
- **专用数据通路**：每个关键部件之间用专用的连线直连而不是共享总线——这是控制最简单的硬布线选项，但路径固定、扩展困难。💀 现在少用。

**数据通路的时序控制** 是数据通路的「时钟叠加层」。一个时钟周期完成一组原子微操作（如将某寄存器的值输出到总线），下一周期不同部件采样该结果并存入目标寄存器。控制器通过按节拍选择性地使能锁存器/缓冲器，按精确的时钟推进。

### 指令执行的数据通路分析

408 大题最喜欢出的题目是——给一个简化的 CPU 数据通路图，让你写出 ADD、LOAD、STORE、JMP 等几类典型指令在每一个时钟周期中数据通路的具体步骤。这是本章最大的分值来源。

以下以单总线数据通路为例，展示四类核心指令的执行流程（PC 的取指后自动自增已预设）：

**取指令（所有指令的公共第一步）** 🔥：
1. PC → MAR（地址送存储器地址寄存器）
2. M(MAR) → MDR（从内存读出指令代码）
3. MDR → IR（指令送入指令寄存器）
4. PC + 1 → PC（PC 指向下一条顺序指令）

**运算类指令（如 ADD R1, R2）** 🔥：
1. R1 → ALU；R2 → ALU（将两个通用寄存器的值打入 ALU 输入）
2. ALU 做加法
3. ALU 输出 → R1（结果回写到目标寄存器 R1）
（注意 RISC 型只能用寄存器，访存步骤不出现）

**访存类指令（LOAD R1, addr / STORE R1, addr）** 🔥：
- LOAD：
  1. addr → MAR（地址由指令给出 → 送到 MAR）
  2. M(MAR) → MDR（内存读出数据）
  3. MDR → R1（数据存入目标寄存器）
- STORE：
  1. addr → MAR
  2. R1 → MDR（寄存器数据送入数据缓冲区）
  3. MDR → M(MAR)（写入内存）

**转移类指令（JMP addr / JZ addr）** 🔥：
1. addr → PC，直接覆盖 PC
（条件跳转 JZ 先检查 PSW 的 ZF 标志——为 1 才执行 addr → PC，否则不做操作继续顺序执行）

## 控制器的设计

### 硬布线控制器（组合逻辑）

硬布线控制器 🔥 用纯粹的**组合逻辑电路**产生所有微操作控制信号。它的输入包括：指令操作码（来自 IR）、当前节拍序号、状态标志（PSW 的位）。输出就是一系列使能信号——选通哪个寄存器输出到总线、ALU 做什么运算、哪些地址信号需要被释放等。

实现过程分两步：首先根据每类指令的要求列出每一步的微操作序列（如 T1 执行 `PC→MAR`，T2 执行 `MDR→IR` 等），然后对每个微操作信号写成一个组合逻辑函数（是输入信号与节拍的布尔表达式）。最后用门电路的阵列实现所有表达式。优点是速度快（信号只需经过若干逻辑门延迟），缺点是设计复杂、不易修改和扩展。

> 🔥 绝大多数 RISC 处理器（ARM、MIPS、RISC-V）使用纯硬布线控制，才能把时钟频率推到几千兆赫。硬布线是高频 CPU 的必选项。

### 微程序控制器

微程序控制器 🔥 把每条机器指令的执行变成一个**微程序的执行**。微程序是一系列**微指令**的序列，每一条微指令在一个时钟周期内控制数据通路完成一组并行的微操作。微程序存放在 CPU 片内的**控存（控制存储器）**中——通常是 ROM。

**微指令格式**分两种：
- **水平型微指令** 🔥：微指令的每一位直接对应一个微操作控制信号——控制信号直接用位映射，1 表示使能、0 表示禁止。好处是并行度高——一条微指令一次可同时触发大量控制信号，速度快。缺点是微指令字很长、控存容量占用大。
- **垂直型微指令** 💀：用一小段编码表示一个微操作类型——类似机器指令的小型码，经过译码再产生控制信号。微指令短得多，但并行度差且需增加译码延迟。已很少单独使用。

**微程序控制器工作过程**：取指周期后，指令的操作码经过**微地址转换机构**映射到该指令微程序在控存中的首条微指令地址。然后控制器逐个读取微指令并执行，直到遇到微程序的「END」标记，回到取指状态等待下一条指令。

**微地址形成方式**：
- **增量法**：顺序执行微程序，默认下地址 = 当前微地址 + 1。遇到微程序分支时需额外逻辑决定。
- **断定法** 🔥：下地址直接在微指令中指定（字段中存下一微地址）。灵活性高——条微指令可以跳转到控存任何位置，微程序无需顺序排列。考研涉及较多。

**微命令的编码方式**：对水平型微指令，要对控存中的控制位进行分组编码——互斥的微操作可以共用一组编码字段，非互斥的各自独立占位。编码方式决定控存宽度和实现复杂度。

> 🔥 Intel x86 从 Pentium Pro 起通过「微操作」把前端 CISC 指令转译成后端的 RISC 风格微操作再派送给硬布线流水执行。这也是微程序思想在现代 CPU 中的遗存——**不是纯微程序也不是纯硬布线，而是两者搭配**。

## 指令流水线

### 指令流水线的基本概念

指令流水线 🔥 🔥 是实现指令级并行的硬件核心技术，类似工厂装配线——把每条指令的处理拆分为多个固定阶段（如常见的五段 IF→ID→EX→MEM→WB），每个阶段由一个独立的流水段完成。各段可同时处理不同的指令——第一段在取第一条指令时第二段为空，下个周期第二条指令就可进入 IF 的同时第一条指令进入 ID 并进行重叠执行。理想条件下每周期可完成一条指令。

> 五段经典流水线 🔥：**取指 IF → 译码/读寄存器 ID → 执行/计算地址 EX → 访存 MEM → 写回 WB**。这是 RISC 流水线的默认模型，408 所有流水线计算都基于此。

### 流水线的性能指标

- **吞吐率（Throughput）** 🔥：单位时间内完成的指令条数。理想 n 段流水线每周期完成一条指令——吞吐率 = 频率（如果无冲突）。408 大题常要计算实际吞吐率（考虑冲突停顿后的平均完成率）。
- **加速比（Speedup）** 🔥：非流水线耗时 ÷ 流水线耗时。理想 n 段等长流水线加速比为 n（第一条填满流水线完成后每周期出 1 结果）。实际加速比远低于 n，因为存在冲突间隙。
- **效率**：流水线各功能段忙碌的时间占总时间的比值——反映硬件利用率。空闲越多效率越低。

考试公式（背熟）：

> 设流水线有 k 段，每段时间 Δt（假设等长），执行 n 条指令：
> - 串行耗时：T_serial = n × k × Δt
> - 流水耗时：T_pipe = k × Δt + (n−1) × Δt（填满第 k 条后每 Δt 完成一条）
> - 加速比 S = (n×k) / (k + n − 1)，当 n>>k 时趋近于 k

**实际流水线多数不等长**——各段的 Δt 不同，瓶颈段决定整条流水线的节拍速率。此时的 T_pipe 需用瓶颈段时间代入为每周期固定时间。

### 影响流水线的因素

流水线的阻碍因素就是三种「相关（Hazard）」🔥🔥——408 年年必考的流水线推理题：

**1. 结构相关**：多条指令在同一个周期争用同一硬件资源。比如 IF 段和 MEM 段都需要访问同一块存储器——如果 Cache 分开（I-Cache/D-Cache）能化解 A。如果不可避免则含资源冲突的硬件需要插入暂停。

**2. 数据相关**：不同指令间的数据依赖性引发冲突。在五段流中典型数据冒险：
- **RAW（Read After Write）**：指令 j 试图读一个还没有被指令 i 写入的寄存器值——这是唯一真实产生停顿的依赖。
- WAR 和 WAW 在乱序执行流水线上才出现——简单的五段按序流排不存在 WAW/WAR 风险。

**旁路技术（Forwarding / Bypass）** 🔥：最强数据处理方式——不等指令 i 写回寄存器，直接将 ALU 的计算结果在 EX 阶段输出反馈到后一条指令的输入。解决了大多数 RAW 冲突，不需停水。

**3. 控制相关**：遇到分支/跳转指令——下一条指令不知道会从哪取。分支判定通常在 MEM 或 EX 期才完成，滞留等待即浪费多拍。解决方案：
- **分支预测** 🔥：预测方向是跳转 K？现代 CPU 采用分支预测器（BTB）和预测带宽缓存预测跳转目标地址提前取指令序列。
- **延迟槽** ⚠️：MIPS 的策略——分支后面跟着一条保证一定执行的延迟指令，分支判决前先拿这条干活，避免空转。现在很少再用（因为流水级增厚后高延迟让延迟槽复杂到不可管理），考研了解历史即可。

### 流水线的分类

**指令流水线**是最基本的——指令层面重叠。此外还有**算术流水线**（如浮点乘除法的流水化），以及更高层的**系统流水线**。考研只深入考查指令流水线，后面的超标量已是进阶。

### 超标量流水线

超标量流水线 🔥 在单核上实现「多发射」——每个时钟周期能同时发射多条指令进入多重脉动线并行处理。这依靠 CPU 内部有多套复用的功能单元（多 ALU、多访存、多浮点单元）和指令调度电路。现代 CPU 都是 4–8 宽度发射的超标量设计，加上乱序执行可获得极高速率。

408 对超标量的考核范围限于了解概念和与普流水线的吞吐差值，不会出具体发射宽度优化计算。

> 🔥 现代 x86 和 ARM 全都是超标量+乱序执行处理器，但它们的原理远超考研 408 范围。

## 异常和中断

### 内部异常和外部中断

所有打断 CPU 正常执行的因素分为两种：

**内中断（异常）** 🔥 🔥：由 CPU 指令执行直接引发——是同步事件。再分三类：
- **故障（Fault）**：是可能被修复的异常，处理完后返回该条引起错误的指令重新执行（如缺页异常）。
- **自陷（Trap / 软件中断）**：是故意安排的——为了由用户态进入内核态发出系统调用（如 x86 的 INT 80 指令）。
- **终止（Abort）**：严重的硬件错误（如非法指令、总线错误），无法恢复——只能终止进程。

**外中断（中断）** 🔥 🔥：由 CPU 外部设备发出请求、异步于指令执行——包括可屏蔽中断（INTR，由中断允标志 IF 控制，可被阻塞）和不可屏蔽中断（NMI，灾难信号——必须立刻处理）。

### 中断处理过程

标准中断处理六部曲（408 必须倒背）：

1. **中断请求**：外部设备通过 INTR/NMI 线向 CPU 发请求。
2. **中断判优**：多个请求中选优先级最高的一个（硬件优先编码器或软件轮询）。
3. **中断响应** 🔥：CPU 在执行完当前指令后检测中断请求线。如果中断允许且优先级高于当前程序——进入中断周期。CPU 硬件自动将**断点（当前 PC + PSW）压入系统栈**，然后从**中断向量表**获取对应中断服务程序的入口地址 → 装入 PC 转到中断处理。
4. **中断服务**：执行中断处理程序（设备相关操作）。
5. **中断返回**：中断服务程序最后执行中断返回指令（IRET），从栈中弹出之前保存的 PC 和 PSW，恢复主程序的执行环境。

**中断向量表** 🔥 是一个存放各中断服务程序入口地址的内存区域，CPU 在硬件识别出中断类型后自动按类型号 x 寻址表中第 x 条入口地址。

## 常见题型

- **数据通路分析**（大题核心）：给 CPU 的通路图，写出 ADD、LOAD、STORE、JMP 每节拍的微操作序列。注意不同序结构要多写访存数据流程。
- **流水线的时空图与性能计算**（大题高频）：画流水线填满的过程，标明冲突暂停和旁路转发。计算吞吐率、加速比、效率。注意不等长流水需按瓶颈周期算。
- **流水线冲突分析**：指出给定指令序列的哪种数据相关和数据冒险类型，并提出旁路/暂停解决。
- **微程序控制器的微指令序列**：给某指令的操作步骤，写出每条对应微指令的硬件位映射——主要考查水平型微指令的并行微操作的编码与位分配。
