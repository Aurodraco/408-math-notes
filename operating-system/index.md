# 操作系统概述

> **考情提示**：本章内容偏概念，408 中一般 1–2 道选择题（2–4 分）。重点：操作系统的四大特征、并发与并行的区别、中断与异常的处理、用户态/核心态的切换、宏内核与微内核对比。知识点零散但难度不高，属于「送分题」。

## 操作系统的基本概念

### 操作系统的定义

操作系统（OS）🔊 是整个计算机系统中最底层、最核心的系统软件，它作为**硬件与用户/应用程序之间的接口层**。OS 做的事只有三件：第一，**控制和管理整个计算机系统的硬件和软件资源**——CPU、内存、磁盘、外设通通归它调度；第二，**合理地组织、调度计算机的工作与资源的分配**——让多个程序能和平共存、各取所需；第三，**为用户和其他软件提供方便的接口和环境**——屏蔽硬件细节，应用程序只需要通过系统调用来请求服务。

这三句话是操作系统考卷中一切具体知识点的根源。对用户来说 OS 是一个「虚拟机」——让每道程序都感觉自己独占整个计算机；对硬件来说 OS 是一个「管理者」——分配资源的裁判员。

### 操作系统的特征

操作系统区别于普通软件的四大核心特征 🔥（408 年年必考）：

1. **并发** 🔥 🔥：两个或多个事件在**同一时间间隔内**发生——这是操作系统最基本最重要的特征。注意**并发 ≠ 并行**——并发是逻辑上同时发生（宏观上看起来一起进行），并行才是物理上的真正同时执行（多核 CPU 上发生）。当系统只有一个 CPU 时，多任务是并发而非并行。408 在此喜欢单选题：「单核 CPU 上运行多个进程是【并发/并行】」——答案是并发。

2. **共享（Sharing）** 🔥：系统中的资源可被多个并发进程共同使用。共享分两种：
   - **互斥共享**——每时刻只能一个进程访问该资源（如打印机、临界资源），必须配合同步机制确保不冲突。
   - **同时共享**——多个进程可以同一时间段交替访问该资源（如磁盘），但微观上各时刻还是只有一个在访问。共享是对资源的高效利用。

3. **虚拟（Virtualization）** 🔥：把物理上的一个实体映射为多个逻辑上的对应物。如虚拟内存把有限物理内存虚拟成每个进程都拥有一个完整的地址空间；虚拟 CPU 让每个进程觉得自己占有整个处理器。虚拟化是现代操作系统的灵魂技术——云计算和容器化的基石。

4. **异步（Asynchronism）**：程序的执行不是等号前的函数调用有序进行，而是以不可预期的速度推进——中断随时可以来、进程可能随时被切换。操作系统必须保证多次运行同一程序在相同输入下总可以得到相同结果，尽管内部过程千变万化。

### 操作系统的目标和功能

OS 的核心目标和功能排列为三大块：

**资源管理** 🔥：OS 负责管理四类资源——处理机管理（进程调度）、存储器管理（内存分配与保护）、设备管理（中断/DMA/缓冲/设备分配）、文件管理（外存空间管理、目录结构、文件保护）。这四大功能直接对应本教程后续的四章内容。

**作为用户与计算机硬件之间的接口** 🔥：OS 向上提供两类接口——**命令接口**（联机命令接口和脱机批处理命令）和**程序接口**（系统调用编程接口的 API——如 Windows 的 WINAPI 和 POSIX 标准中的系统调用）。考研程序只关心程序接口（系统调用）。

**实现了对计算机资源的扩充**：在物理硬件之上通过软件构建出更高级更易用的抽象。比方说文件就是磁盘上数据块的抽象——程序员永远不需要了解磁盘扇区号，只需要 `open()`、`read()`、`write()`。

## 操作系统的发展与分类

### 手工操作阶段 → 批处理阶段

最早的计算机没有任何操作系统——由程序员通过插入纸带或拨动开关直接控制机器。人机矛盾——CPU 远快于手工干预——催生了最初的批处理思想。

**联机批处理**：操作员把一批作业按先后顺序排列在批输入设备上，由调度程序串行加载执行。每次作业的 I/O 还占用了 CPU 的等待时间——CPU 在这种情形下利用率极低。

**脱机批处理** 🔥：利用卫星机将作业从低速输入设备转存到高速磁带，再将磁带接入主计算机高速输入。CPU 从高速磁带上直接读取作业，输入/输出工作从主机分流到卫星机——这是现代 I/O 控制器和 DMA 的思想雏形。

### 多道批处理系统

多道批处理系统 🔥 🔥 将 OS 发展推入了一个质变阶段——内存中同时驻留多道程序，当一道程序因 I/O 操作被阻塞等待时，CPU 马上调度到另一道可以执行的程序。多道批处理的本质是**让 CPU 和 I/O 设备并行工作**，大幅度提高资源利用率。但交互性极差——用户提交作业后只能等着结果出来，过程中无法干预。

### 分时操作系统

分时操作系统 🔥 是多道批处理的交互性改良版。CPU 的时间被切成极短的时间片（time slice），轮转分给各个在线的交互用户——每个用户觉得自己独占整台计算机。分时系统实现了「人机交互」这一巨大的体验突破—用户的击键/鼠标操作能得到及时响应。代表：Unix 🔥🔥——现代操作系统的标杆，在此基础上衍生出 Linux 和 macOS。

### 实时操作系统

实时操作系统 🔥 不追求周转率，而追求**在规定时间内一定完成规定的动作**——对响应时间的确定性有硬指标要求。分**硬实时**（必须在截止时间内完成，否则灾难后果——如导弹控制、反应堆控制）和**软实时**（偶尔超时仍可以——如多媒体播放）。实时系统广泛应用于嵌入式设备 🔥。

### 网络操作系统和分布式操作系统

**网络操作系统** 🔥：在单机 OS 上增加了网络通信能力——各计算机仍自治运行，通过如 NFS（网络文件系统）等协议共享一些资源。Windows NT 和大多数现代 Linux 发行版都属于网络 OS。**分布式操作系统** 🔥 更进一步：整群计算机在地理上分开但逻辑上对用户表现为一台单一的系统——资源调度和任务分配在分布式 OS 统一集中管理。当前云计算和容器编排系统是分布式 OS 思想的商业级体现。

> 408 只需区分这两个术语的概念——网络 OS = 各机独立 + 网络通信；分布式 OS = 多机逻辑合一、统一调度。未出过深层大题。

### 个人计算机操作系统

个人计算机操作系统是单用户多任务形式——用户独享整台机器，但系统同时支持多个任务并发运行。Windows、macOS 和桌面 Linux 都是此类。这部分 408 仅概念了解——不深究。

## 操作系统的运行环境

### 处理机的执行状态

为了解决对系统资源的保护问题，CPU 至少提供两种运行态 🔥——这是一个纯硬件的状态位：

- **用户态（目态）** 🔥：应用程序运行的状态。只能执行受限的指令集（非特权指令）。不能直接访问硬件资源、不能修改敏感的 CPU 控制寄存器、不能切换状态。用户态的代码如试图执行特权指令，CPU 会引发中断陷入内核使操作系统介入处理。
- **核心态（管态/系统态）** 🔥 🔥：OS 内核代码的运行状态。可以执行全部指令（含特权指令）——直接操控硬件、修改状态寄存器、访问所有内存空间等。

运行态和操作系统对进程的受信管理同生共存——内核态是安全边界，进程只有通过系统接口（系统调用）才能请求内核代为执行特权操作，这个转换机制就是**陷入（Trap）**。内核态开销很高（状态切换保存现场）,所以现代操作系统缩减内核态的总耗时。

> 408 选择题高频考点：「哪个操作可以使用户态切换到核心态？」答案：系统调用、中断、异常（三种）。「可以在用户态运行的代码是什么？」答案是应用程序本身——不能执行 I/O 端口操控和修改页表等操作。

### 中断和异常

中断是所有异步事件的总称——发生在任何指令执行间隙，打断 CPU 正常程序流转。分两大类 🔥：

- **中断（外中断）**：CPU 外部的事件信号（如 I/O 完成、定时器超期、用户按下键盘）。它来自硬件控制器，在每条指令结束时检查。
- **异常（内中断）**：CPU 内部指令执行中产生异常——同步于指令执行。分为「故障」（缺页）、「自陷」（系统调用指令主动引发的软中断）和「终止」（硬件灾难性错误，如总线错误、奇偶校验错）。考研常出现在组成原理章，但在操作系统里也会复考。

**二者的区别**：中断 = 外来的异步；异常 = 内部的同步——且在导致中断的指令结束前就能检测并处理。

### 系统调用

系统调用 🔥 🔥 是用户程序与操作系统内核之间的**唯一合法入口**。所有对硬件的访问和对系统资源的使用都必须经过系统调用——如设备管理、文件操作、进程创建和通信、内存申请和释放等。

系统调用的执行过程涉及 CPU 状态切换：
1. 用户程序调用库函数（C 库 `open()` 等）——该库函数通过一条特殊指令（x86 中 `INT 80` / `syscall`）从用户态陷入核心态。
2. CPU 硬件自动保存用户态的 PC 和 PSW 到系统栈，然后根据系统调用号在系统调用表中寻找对应的内核服务程序的入口地址。
3. 内核执行相应服务 —— 完成后恢复用户态现场并返回用户程序继续执行。

**系统调用与库函数的区别**：库函数运行在用户态，用来封装系统调用（如 `printf()` → 库函数做了格式化后调用 `write()` 系统调用发送输出）。考研中选择题常考「以下哪个是系统调用」——`open()`、`read()`、`write()`、`fork()` 是系统调用；`printf()`、`scanf()` 是库函数。

## 操作系统结构

### 分层法

分层法 🔥 按功能将 OS 从上到下分成若干层次，每层只使用下一层提供的接口。底层是硬件，顶层是用户接口——结构清晰易于差错和调试。缺点是层层委托开销大、层定义不易。

### 模块化

模块化 🔥 将 OS 按功能分解为若干独立的模块——模块间通过明确的接口通信。模块化是实际 OS 中使用最多的组织方式（Linux 的模块可动态加载和卸载）。优点：灵活、可维护性和可扩展性强。缺点：模块间接口定义不当容易耦合严重。

### 宏内核 vs 微内核

| 对比项 | 宏内核 🔥 | 微内核 🔥 |
|--------|----------|----------|
| 功能位置 | 所有 OS 核心服务（文件系统、网络协议栈、驱动）都在内核态运行 | 仅最底层的 CPU 调度、内存管理、IPC 在内核态，其余服务移到用户态 |
| 调用方式 | 直接内联函数调用——高效 | 用户态服务进程通过 IPC 消息传递——开销大 |
| 效率 | **高**（函数调用零开销） | 低（频繁消息传递导致上下文切换） |
| 可靠性 | 单一内核模块 bug → 全系统崩溃 | 模块隔离——各服务独立崩溃可重启 |
| 代表 | **Linux** 🔥🔥、Windows | Minix、QNX、华为鸿蒙微内核 |

> 🔥 现代 OS 大都是带可加载模块的宏内核——Linux 所有驱动和文件系统在内核态但可灵活加载和卸载。微内核因为 IPC 性能瓶颈，市场份额最终一直没有超过宏内核。

> 408 选择题反复出现宏/微内核特征的区别题。口诀：**宏快不稳微安全慢**。

### 外核

外核（Exokernel） 💀 是一种极端的极简内核——把物理计算机资源直接划分后分别映射给每个运行在用户态的管理程序，取代传统内核的抽象层。各应用程序可以按需实现自己的资源管理策略——高效但极其复杂。外核对于考研完全是兴趣阅读量——知道「内核的更极端简化形式」即可。

## 常见题型

- **操作系统的四大特征**：并发 ≠ 并行、共享分互斥和同时、虚拟技术创造逻辑资源、异步通过信号/中断处理。
- **中断和异常的处理过程**：CPU 在每条指令末尾检查中断线，响应过程依次为关中断、保存断点、从中断向量获取服务程序地址、执行完毕恢复现场。
- **系统调用的执行流程**：用户态程序 → 陷入指令 → 核心态 → 系统调用表查内核函数入口 → 内核处理 → 恢复现场返回。
- **宏内核与微内核的对比**：考卷上反复出现——宏内核功能全在内核（快但不稳），微内核基本只留最小内核其他在用户态（稳但慢）。
