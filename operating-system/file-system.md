# 文件系统

> **考情提示**：文件系统在 408 中分量中等，通常 1–2 道选择题 + 偶尔一道混合索引计算大题（≈10 分）。重点：混合索引地址范围和盘块号计算、位示图分配与回收、磁盘调度算法寻道计算。知识点比较零散但有清晰的公式和套路。

## 文件系统基础

### 文件的概念

文件（File）🔥 是存储在辅助存储介质上的**具有符号名称的信息集合**——是操作系统对磁盘上以逻辑的方式呈现数据的基本单位。每个文件都有其属性集合：名称、类型、大小、位置（在存储设备上的物理地址）、创建时间和最后修改时间、访问权限等。文件属性存放在文件目录项中。

### 文件的逻辑结构

文件的逻辑结构是从用户视角看到的数据组织方式：

- **无结构文件（流式文件）** 🔥：以字节流的方式组织——没有明显的的记录边界。源程序、可执行文件、文本都是流式文件。现代 OS 的默认文件都是流式文件。
- **有结构文件（记录式文件）** ⚠️：由若干记录组成，记录间有逻辑关系。再细分四型：
  - **顺序文件**：记录按键顺序排列——顺序存取快但搜索慢。
  - **索引文件**：建立关键字-记录位置的索引表——和数据结构索引查找原理一致。
  - **索引顺序文件**：记录顺序存成分区，每区再建索引——ISAM 经典结构。
  - **直接文件 / 哈希文件**：关键字通过哈希函数直接转换为记录地址。

> 💀 记录式文件在现代通用 OS 中基本不用——关系数据库等上层软件自己实现。408 了解分类，不会深进代码。

### 目录结构

目录就是「文件名→文件属性/FD 的映射表」，五种进化过程：

- **单级目录** 💀：整个系统一个线性表——不能有重名文件，搜索速度慢。
- **两级目录**：每个用户拥有独立的文件目录——解决重名和一定的访问控制。
- **树形目录** 🔥 🔥：根目录下有子目录→子目录下再延展多层次——完全无重名冲突。每个进程有一当前目录概念（相对路径 vs 绝对路径）。
- **无环图目录**：允许文件/目录被多个目录项共享——通过硬链接/软链接实现（见下节「文件共享」）。
- **目录的操作**：创建/删除/打开/读目录/重命名/链接和解除链接等。

### 文件共享

文件共享的核心目的是让多个用户或多个进程能共同访问同一文件：

- **硬链接（Hard Link）** 🔥：同一文件由多个目录项指向——基于**索引结点（inode）**。在文件 inode 里有一个链接计数（link count）记录有多少目录项指向它——当 link count 归零时物理块才会被释放。硬链接只能在同一文件系统内（索引结点不跨文件系统），也不能对目录创建（防止环路除非系统支持）。
- **软链接（符号链接 / Symbolic Link）** 🔥：一个特殊文件保存另一个文件的路径名。通过路径重解析定位真实文件——删除原文件后软链接沦为「断开的悬空链接」。可跨越文件系统，使用灵活但多一次路径解析。

> 🔥 Linux 中 `ln` 默认创建硬链接，`ln -s` 创建软链接。Windows 中的快捷方式就是软链接思维的体现。

### 文件保护

文件保护分三级：
- **口令保护**：在打开文件时需要提供口令——实现简单但口令存在于系统密文比较脆弱且单一口令管全部。💀 安全性弱。
- **加密保护**：对文件数据加密存储，只有持有正确密钥才能访问——安全性极高但加密开销重。🔥 对敏感数据使用。
- **访问控制** 🔥 🔥：最通用方案——每个文件拥有访问控制表（ACL），记录哪些用户/组具有怎样的权限（读/写/执行/删除）。现代 OS 标配权限管理均基于 ACL。

## 文件系统的实现

### 文件的物理结构（文件的分配方式）

这是文件对磁盘块的「用什么形式放置」的核心问题 🔥：

**连续分配** 🔥：每个文件在磁盘上占用一组**物理上连续的盘块**。存取速度快（顺序寻访不需跳来跳去），但易产生外部碎片，文件无法动态增长——必须预先指定最大大小。💀 纯连续分配现在很少用。

**链接分配** 🔥：
- **隐式链接**：每个盘块包含一个指下一盘块的指针——顺着链定位。缺点是没有随机访问能力——读一个文件最终需从头到尾遍历。💀 很少使用。
- **显式链接（FAT）** 🔥 🔥：把链指针统一存放成一张**文件分配表（FAT）**——放在磁盘首部。目录项只放文件的起始块号，之后的每块由 FAT 表链起访问下一块。FAT 表支持随机存取（沿着 FAP 链跳转），并且动态增删方便（链表插入在表中操作）。**FAT32/exFAT 至今广泛用于 U 盘和 SD 卡** 🔥。

**索引分配** 🔥 🔥：每个文件拥有一个独立的**索引块**专门存放该文件所占的全部盘块号列表。
- **单级索引**：索引块直接存所有盘块号。文件大小受限于索引块大小÷指针大小。
- **多级索引**：索引块指向更下一级的二级索引块再指盘块数据——大到能达到很大容量但增加了一层块的查询。
- **混合索引** 🔥 🔥：如 UNIX/Linux 的 inode——前 10‒12 个直接块存小文件盘块号，再配加间址的间接块三级——一级间接块、二级间接块、三级间接块依次加倍步高度增大文件最大容量。408 大题的高频计算公式：给定块大小指针字节和索引级数 → 计算出文件最大字节。

### 文件存储空间管理

**空闲表法**：维护一张空闲块号表——连续的空闲块一行为一个记录。分配回收类似于连续分区分配。

**空闲链表法**：用空闲块的第一个字节做指针链接所有空闲块成一条链。分配拿出表头，回收画回头部。

**位示图法** 🔥 🔥：最常用也是最易考大题的方法。用一个位向量表示所有块的使用状况——每个块对应一位（0 空闲 / 1 占用）。计算块号与位图行列的相互转化可通过简单矩阵下标映射实现。位示图占用极少的固定空间（8 个字节可表示 64 个块的状况），因此被现代多数系统选为磁盘管理描述结构。

**成组链接法** 🔥：UNIX/Linux 的实际文件系统空闲管理方案——将空闲盘块号分组存链，类似链表与表结合的布局，能快速分配回收大量块。

> 408 位示图绝对是最高频考题——给位图比特位字串和要分配的/回收的块信息，要求更新位示图并找出对应盘块号和行/列的字位置。

## 磁盘组织与管理

### 磁盘的结构

磁盘 🔥 的物理寻址参数 CHS：
- **柱面（Cylinder）**：同一半径的所有磁道形成柱面——相同柱面的数据不必移动磁头就可连续访问
- **磁头（Head）**：每个盘面有一个读写磁头——头号等于盘面号
- **扇区（Sector）**：磁道上分为若干扇区——是磁盘的最小读写单位
- **磁盘的物理地址** = (柱面号, 磁头号, 扇区号) 即 CHS。现代磁盘向 OS 展现的 LBA（逻辑块地址）简化了这些内部几何。

### 磁盘调度算法

磁头寻道是磁盘 I/O 最大的延迟来源（毫秒级），效率极大地取决于磁头寻道调度的次序。408 要求对以下五种算法逐步推算出平均寻道长度 🔥：

| 算法 | 要点 |
|------|------|
| **FCFS** | 按请求到达顺序调度——最公平，寻道路径可能大幅往复跳跃，效率低。|
| **SSTF** 🔥 | 优先距当前位置最近的柱面请求——寻道距离最优，但「饿死」远处请求——不长久。|
| **SCAN（电梯算法）** 🔥 | 磁头像电梯——从内圈到外圈扫描处理一路上的请求，到头再反方向扫描。避免了饥饿。|
| **C-SCAN** 🔥 | 磁头只单向扫描，走到尽端立刻跳回另一端重新开始扫描——请求等待时间更均匀。|
| **LOOK 和 C-LOOK** 🔥 | 和 SCAN/C-SCAN 相同，但磁头只走到最远的请求端立即回折而非物理最尽端——实质改进现实所用。|

408 的答题套路：给当前磁头方向和位置 + 一串等待请求柱面，对每个算法计算磁头移动的顺序和总寻道距离。**LOOK 系列和对应的 SCAN 结果末尾段会有差别**——行走的最后一个段不是到物理尽头端。

### 磁盘读写时间

一次磁盘 I/O 全程时间 = 三个部分的累加 🔥：
1. **寻道时间（Seek Time）**：移动磁臂到目标柱面的机械时间——是最大的开销（4‒12ms）。
2. **旋转延迟时间（Rotational Latency）**：等待盘面旋转到目标扇区下——正好转要半圈的均时 = 1 / (2 × 转速)。
3. **传输时间（Transfer Time）**：数据实际读写的时间——远短于上面两项（远小于 1ms）。

> 408 可能要求一个算平均值：一次 I/O 时间 = 寻道时间 + 平均旋转延迟 + 传输时间。需能套公式。

### 磁盘管理

- **磁盘格式化**：低级格式化建立磁盘表面结构和起始地址信号；文件系统格式化创建超级块、空闲位表、根目录等首元数据。
- **引导块**：磁盘的扇区 MBR 上有一段启动引导代码——ROM BIOS 首先读它执行，再由它引导 OS。
- **坏块处理**：制造时产生的坏扇区记录在缺陷表中不分配——逻辑上用隐藏在备用扇区透明替换。

## 常见题型

- **混合索引的地址计算（大题重中之重）**：给 inode 结构（直接块间接块级数）和各块大小→算出文件最大容量及给定偏移的直接块与间接块层数分析。
- **位示图的盘块分配与回收**：给位示意图的字大小和块号→计算出该块对应的字号和位号并写出分配或回收后的值在 16 进制表示。
- **磁盘调度算法的寻道计算**：一律给出当前磁头位置方向加请求链——一个个写磁头移动顺序并计算总寻道距离。
- **FAT 表大小计算**：给盘块数和表项位数→FAT 表占用的存储量（块数量×表项大小）；加上支持单表本身大小在其他存储中的占用。
