# 进程管理

> **考情提示**：进程管理是操作系统最核心的章节——PV 信号量操作（大题几乎年年必考 10 – 15 分）+ 调度算法计算（选择题高频）+ 银行家算法（选择题/简答题经常出现）。本章是你 408 操作系统得分的基本盘。

## 进程与线程

### 进程的概念和特征

进程 🔥 是程序在给定数据集上的一次执行过程——**系统进行资源分配和调度的独立单位**。进程由三个实体组成：**进程控制块 PCB**（操作系统用来管理进程的数据结构，存放进程标识符、状态、优先级、程序计数器、寄存器映像、内存分配信息、打开文件表等所有进程信息）、**程序段**（指令代码）和**数据段**（运行时变量、堆、栈）。

进程的四大特征（与 OS 四大特征一一对应）：
- **动态性**：进程是程序的一次执行——有创建、活动、暂停和结束的生命周期。
- **并发性**：多个进程可在同一时间间隔内同时运行。
- **独立性**：进程是独立获取资源的实体。
- **异步性**：各个进程按各自的不可预知的速度向前推进。

> 408 核心辨析：**进程与程序的区别**——程序是静态的指令序列（存在磁盘上的可执行文件），进程是动态的执行实例（在内存中有 PCB 的那个）。一个程序可以对应多个进程（如多次打开同一浏览器），一个进程只能执行一个程序。

### 进程的状态与转换

经典**五状态模型** 🔥（408 必须画熟整个状态转换图）：

1. **创建态（New）**：进程被创建尚未加入就绪队列——OS 正在分配 PCB、装入程序、初始化资源。
2. **就绪态（Ready）** 🔥：进程已得到除 CPU 外的所有必要资源，只等调度器分配 CPU。
3. **运行态（Running）** 🔥：进程正占用 CPU 执行指令。单 CPU 系统同时最多一个运行态进程。
4. **阻塞态（Blocked / Waiting）** 🔥：进程因等待某事件（I/O 完成、资源可用、收到信号等）暂时无法继续。
5. **终止态（Terminated）**：进程正常结束或异常终止，OS 正在回收 PCB 和资源。

状态转换路径：创建 → 就绪（创建完成）→ 运行（调度选中）→ 终止（运行结束）。运行 → 就绪（时间片用尽或高优先级抢占）。运行 → 阻塞（请求无法立即可用的资源）。阻塞 → 就绪（等待事件发生）。

**七状态模型**（含挂起）：在五态基础上新加两个挂起态——**就绪挂起**和**阻塞挂起**。当内存不足时，OS 可以将部分进程从内存对换到外存（swap out），此过程就进入挂起态。挂起状态的进程即使满足运行条件也不能到 CPU 上——必须被先换入内存再做调度。这是中级调度的依据。

### 进程控制

进程控制用**原语（Primitive）** 🔥 完成——原语是在内核态下运行的不可中断的最底层过程，通过硬件的关中断/开中断指令保证执行的原子性。核心原语包括：

- **创建原语（Create）**：申请空白 PCB → 为进程分配资源 → 初始化 PCB → 将进程加入就绪队列。
- **撤销原语（Destroy）**：回收进程所有资源 → 释放 PCB → 从队列移除。
- **阻塞原语（Block）**：将当前进程从运行态移入阻塞队列，之后调度下一就绪进程运行。
- **唤醒原语（Wakeup）**：将一个被阻塞进程从阻塞队列移入就绪队列（不立即上 CPU）。
- **切换原语（Dispatch）**：保存当前进程的上下文到 PCB → 从就绪队列中选择一个新进程 → 把其上下文从 PCB 恢复到 CPU 寄存器 → 运行。

> 408 问的最多的一句话就是「进程切换何时不能发生」，答案是操作系统内核在执行临界段关键操作时必须屏蔽中断——此时不能响应调度申请。

### 进程的通信

多个协作进程之间经常需要交换数据或多进程互操作。通信机制有三类：

- **共享存储** 🔥：OS 在内存中分配一个共享空间——多进程将其映射到各自虚拟地址空间中。进程直接通过对空间里的共享变量读写进行通信——同步要由程序员处理。是效率最高的 IPC 方式🔥。
- **消息传递** 🔥：操作系统提供 send/receive 原语——无需共享空间。又可按是否只指定一个接收进程区分为**直接通信**（发送给特定目标进程）和**间接通信**（通过共享的 mailbox/端口发收消息）。微内核系统中这是唯一的 IPC 方式。
- **管道通信（Pipe）** 🔥 🔥：特殊的文件方式——连接一个写进程和一个读进程的 FIFO 缓冲区。管道分无名管道（临时）用于父子进程和命名管道（有名 UNIX 中叫 FIFO）用于任意进程间。UNIX 中的 `|` 是一个管道的典型应用。写满读空时都有一个阻塞的行为。

### 线程

线程 🔥 🔥 是在进程内部的一条执行流——每个进程至少有一个线程。线程共享该进程的所有资源（地址空间、文件、信号等），但各自**独有**：线程 ID、PC、寄存器上下文、栈。线程是**CPU 调度的基本单位**——CPU 不再以进程为调度对象而是以线程。

**线程与进程的核心对比**：
| 对比项 | 进程 | 线程 |
|--------|------|------|
| 调度 | 拥有资源的单位 | 是 CPU 调度单位 |
| 资源所有 | 有独立的地址空间 | 共享所属进程全地址空间 |
| 并发代价 | 创建/切换代价大 | 极轻量——创建和切换开销远低于进程 |
| 隔离性 | 独立内存空间互不干扰 | 同一进程中的线程可互相访问内存数据（协作但易竞争） |
| 系统开销 | 进程切换需要完全切换地址空间 | 同一进程线程切换不需换地址空间 |

**线程的实现分三级**：
- **用户级线程** ⚠️：线程库全在用户态工作，内核完全不知道线程的存在——对内核一个进程只有一个调度单元。切换快但任何一个线程堵塞整个进程被阻塞。 💀 逐渐被内核线程淘汰。
- **内核级线程** 🔥：每个线程都由内核管理——调度、阻塞、切换由内核完成。多核 CPU 上同一进程的内核线程可以真正并发执行。Linux 使用内核线程为主。
- **多线程模型**：**一对一**（一个用户线程映射到一个内核线程），**多对一**（多个用户线程映射到一个内核线程——用户并发但内核串行），**多对多** 🔥（用户线程与内核线程非一一绑定——平衡用户灵活性及内核性能）。现代 OS 多用一模型。

## 处理机调度

### 调度的层次

分层调度从宏观到微观分三级 🔥：

- **高级调度（作业调度 / 长程调度）**：决定将辅存中的哪些作业装入主存进入就绪队列。运行频率极低（数秒到数分钟一次）——批处理操作系统独有，现代个人计算机不设作业调度。
- **中级调度（内存调度）** 🔥：决定哪些进程暂时换出到外存（挂起）以腾出内存。根据内存压力触发——频率远高于高级调度。
- **低级调度（进程调度）** 🔥 🔥：决定就绪队列中哪个进程/线程获得 CPU。运行频率最高——几十毫秒一次。这是考研调度算法的主要考察对象。

### 调度的时机

进程调度的发生时机 🔥：① 当前运行进程主动放弃 CPU（结束或阻塞）② 当前进程时间片用完 ③ 有更高优先级的进程进入就绪状态（可抢占时发生）。**不能进行调度的情形**：CPU 正处理中断时非中断进程不能施实调度；操作系统维护临界区数据一致性时关中断期间必须封锁调度。

### 调度的方式

- **非剥夺式调度（非抢占式）**：一个进程拿到 CPU 后一直运行到自身弃权（终止或阻塞）或时间片自然结束。FCFS、SJF 非抢占变种。优点是无上下文切换抢断的复杂——缺点一旦 CPU 被一长作业霸占，后续短作业必须等到天荒地老。
- **剥夺式调度（可抢占式）** 🔥：操作系统随时可从当前进程中剥夺 CPU 给更高优先级或更符合条件的进程。RR、多级队列、多级反馈队列均为此类。现代所有通用系统均采用可抢占调度。

### 调度算法

408 调度算法的核心要求就是对每种算法会手工计算**平均周转时间**和**平均等待时间** 🔥。

| 算法 | 类型 | 抢占 | 核心特点 |
|------|------|:--:|------|
| **FCFS** | 作业/进程 | 否 | 串行先进先出——长作业友好、短作业被堵死。**
| **SJF / SPF** | 作业/进程 | 均可 | 理论上平均等待时间最短——但谁诚实报告所需运行时间？难以预测。长作业可能饥饿。
| **HRRN（高响应比优先）** 🔥 | 作业 | 否 | 响应比 R =（等待时间 + 所需运行时间）/ 所需运行时间 = 1 + 等待/服务。优先调度最高 R 作业——综合补偿短作业和防长作业饿死。是作业调度优选。**
| **时间片轮转（RR）** 🔥 | 进程 | 是 | 固定时间片轮转。关键在时间片大小选择——太大退回 FCFS 无差别，太小上下文切换开销暴增。**
| **优先级调度** | 进程 | 均可 | 可就优先级比较调度。可能发生的饥饿通过增进等待作业的优先级慢慢补偿（aging）。|
| **多级队列** | 进程 | 是 | 多个就绪队列如前台交互队列高优先级、后台批处理低优先级——队列间指定策略队列内其他算法。|
| **多级反馈队列** 🔥 🔥 | 进程 | 是 | 重要内存最综合的设计——时间片可动态延长：新进程进最高优先级短时间片队列，用满 CPU 时间片后降至少优先长时队列。既保证交互响应快又避免了批处理饥饿。现代 OS 的调度思想重要参考。|

> 对这七种算法 408 重点考：**FCFS / SJF / HRRN / RR的周转/等待时间计算 → 选择题；多级反馈队列的巧妙参数变化—→ 简答题。** 练习题优先反复算上几组数据，掌握套公式即可。
