# I/O 管理

> **考情提示**：I/O 管理在 408 内容最少，通常 1 道选择题（2 分）。重点：SPOOLing 技术原理、缓冲区处理时间计算、四种 I/O 控制方式演进。知识点精练、性价比高——花少量时间可拿满本章分数。

## I/O 管理概述

### I/O 设备分类

外部设备的多样性与性能差异巨大，OS 从三个维度分类：

- **按传输速率**：**低速设备**（键盘、鼠标——几个字节/秒）、**中速设备**（打印机——几千到几十 MB/秒）、**高速设备**（SSD/磁盘——几百 MB/秒至几 GB/秒）。
- **按信息交换单位**：**块设备** 🔥（以固定大小数据块为传输单位——磁盘/SSD，支持随机寻址寻块）和**字符设备** 🔥（以字符流为单位——键盘/串口/打印机，不可寻址仅顺序流方式处理）。
- **按共享属性**：**独占设备** 🔥（一次只能分给一个进程的临界资源——如打印机）、**共享设备** 🔥（允许对多进程同时交替访问——如磁盘多个请求可并发调度）、**虚拟设备**（用 SPOOLing 技术将独占设备转化为多个进程的「虚拟」多份独立共享设备）。

> 408 选择题高频区分点：「磁盘属于【块设备/字符设备】？」答案块设备；「键盘属于？」答案字符设备。

### I/O 控制方式

CPU 与 I/O 设备间数据传送控制的四种方式逐步进化以释放 CPU：

- **程序直接控制方式（轮询）** 💀：CPU 不断循环读设备状态寄存器直到设备就绪再执行数据传送。CPU 全被占，效率极低——除极简单的嵌入式环境几乎不用。
- **中断驱动方式** 🔥：CPU 发出 I/O 命令后继续执行其他任务。当设备完成数据准备，通过中断通知 CPU——CPU 进入中断服务程序做数据传送后返回。CPU 与外设并行工作但仍需每传送一字 CPU 介入一次。
- **DMA 方式** 🔥 🔥：DMA 控制器独立完成内存与设备间整块数据的传送——CPU 只在传输前后各参与一次（开始参数设定和结束通知）。是高速块设备（磁盘、网络卡）的实际传送方式。
- **通道控制方式** ⚠️：通道是比 DMA 更强的专门 I/O 处理器——可以独立执行通道程序完成多组 I/O 控制序列完全释放 CPU 对 I/O 的干预。大型机专用——408 了解即可。

这四种方式的演进过程是**从 CPU 全程干预逐步解放 CPU**。408 选择题常给一段描述问是哪种方式，对比点抓「CPU 在每字干预还是整块前后介入还是全程完全不管」。

> 注：DMA 在计算机组成原理已讲解详尽，此处是操作系统视角的配合——操作系统配置 DMA 参数、启动传送、并在结束后响应中断。

### I/O 软件的层次结构

I/O 软件分层 🔥 以隔离硬件差异与系统统一性为目的（从顶到底共五层）：

1. **用户层 I/O 软件**：用户程序中调用库函数 —— 如 `printf()`、`scanf()`。
2. **设备独立性软件** 🔥：实现对所有设备的统一命名、分配、保护和缓冲管理。操作系统在此层提供系统调用接口屏蔽硬件差异。
3. **设备驱动程序** 🔥：对应特定设备芯片的具体控制——由硬件生产商编写。将 OS 的通用读/写请求翻译为该设备特有的寄存器序列。
4. **中断处理程序**：当设备完成 I/O 操作产生中断时，由该层响应中断并唤醒阻塞的驱动层程序。
5. **硬件**：I/O 接口卡片、控制器、物理 I/O 总线等实际电路。

## I/O 核心子系统

### I/O 调度

I/O 调度 🔥 在磁盘访问上最常体现——OS 维护每设备的请求队列，通过调度算法重排 I/O 请求的顺序以优化寻道/延迟性能（如磁盘调度中的电梯算法）。也适用于通用性的 I/O 带宽分配。

### 高速缓存与缓冲区

高速缓存（Cache）和 **缓冲区（Buffer）** 🔥 是两项不同功能但相似实现的技术：缓存保留数据将来可能用到的副本提高读取速度；缓冲区是在不同速度的设备间暂存数据实现二者异步——生产者进程往缓冲区写，消费者消费者从中取。OS 在 I/O 中大量使用缓冲区匹配 CPU 与外设的速度差距。

### 缓冲区

OS 通过缓冲区匹配 CPU 和外设解耦运作 🔥：

- **单缓冲**：进程将要处理的数据先从设备读出到一个系统缓冲区再由进程到缓冲区取。时间处理分析常见于 408 选择题——总时间 = 设备读满缓冲的时间 + 进程处理此块的时间——两者可并行一页部分。
- **双缓冲**：两个缓冲区交替使用——设备在填充一个缓冲时进程正在处理另一个——消除了单缓冲中进程在等读满的停滞。吞吐量加半改。
- **循环缓冲**：多个缓冲区围成一个环——生产者往下一个空缓冲写入、消费者从已满缓冲读取。解决稳定速率匹配。
- **缓冲池** 🔥：OS 维护一个系统公用的缓冲池共享给多进程使用——可有三类队列：空闲缓冲队列、输入缓冲队列（已满等待处理）、输出缓冲队列（待写出外设数据覆盖）。

> 408 经典计算题：某磁盘把一块数据读入缓冲区需要时间 T，进程处理该数据块需时间 C，问 T 和 C 在单缓冲、双缓冲下 CPU 与设备并行关系的甘特图和最长时间。结论：双缓冲在 C < T 时能实现完全并行。

### 设备分配与回收

**设备分配表**包含三张数据结构 🔥（了解概念即可）：
- **SDT（系统设备表）**：系统维护的所有设备属性表。
- **DCT（设备控制表）**：每个设备一张——记录设备状态、等待队列等。
- **COCT（控制器控制表）**和 **CHCT（通道控制表）**：分别用于多层控制时设备的一路级联分配。

设备分配策略要考虑**设备安全性**——分配不当可能造成死锁（进程占用设备 A后请求设备 B、另一进程反向占用 B后请求 A）。

### 假脱机技术（SPOOLing）

**SPOOLing（Simultaneous Peripheral Operations On-Line）** 🔥 🔥 是管理独占设备（尤其是打印机）的经典而至今仍广用的技术。核心思想：用磁盘做一个中间传递缓冲区，把独占型串行物理设备改造成多个进程可「共享」的虚拟化的设备集合。

SPOOLing 系统三部分：
- **输入井**和**输出井**：在磁盘上划定的缓冲区域接受多进程输入数据流/累积输出内容。
- **输入进程**和**输出进程**：负责在井与物理设备间异步传输数据——前台用户进程感觉设备随时可用；后台实际由 SPOOLing 进程按序服务真正设备。
- **SPOOLing 实例——共享打印机**：多个用户并发提交打印请求但不用等待上级设备完成——用户数据→输出井→由输出进程在后台按次序传给打印机逐份打印。用户不知不觉实现了共享和异步打印。

> 🔥 SPOOLing 的思想仍在现代操作系统的打印后台处理、邮件发送后台排队等功能中体现。408 选择题最爱描述打印机共享的场合——答案一律为 SPOOLing 技术。

## 常见题型

- **各种 I/O 控制方式的对比**：按「CPU 参与的字粒度」选是轮询/中断/DMA/通道——选择题排序高频率。
- **SPOOLing 技术的原理**：卷面经典答案要点——「把独占设备改造成可共享的虚拟设备，系统用输入/输出井暂存数据，后台输出进程统一调度设备」。
- **缓冲区处理时间计算**：给读块时间 T 和处理时间 C，在单缓冲与双缓冲下画时间线，问设备利用率、或操作的总完成时间。
- **I/O 软件层次结构**：小题考—应用程序 I/O 函数→设备独立层→驱动程序→中断处理→硬件的调用顺序中，哪个层负责具体的寄存器操作（=驱动程序）。
