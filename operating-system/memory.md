# 内存管理

> **考情提示**：内存管理是操作系统分值最高的章节之一——页式地址变换几乎每年一道大题（≈10 分），页面置换算法缺页率计算也是固定选择题 2 分。本章与计算机组成原理的 Cache/虚拟存储器有大量交叉，学通一科可以互促另一科。

## 内存管理基础

### 内存管理的功能

操作系统内存管理四大核心功能 🔥：
1. **内存空间的分配与回收**：进程创建时分配内存，退出后回收。分配的单位可以是连续分区、页或段。
2. **地址转换**：将程序的逻辑地址转换为对应的物理内存地址——由 MMU（内存管理单元）硬件配合 OS 页表/段表完成。
3. **内存空间的扩充**：通过虚拟存储技术让程序觉得自己拥有比实际物理内存更大的地址空间。
4. **存储保护**：确保每个进程只能访问自己被分配的内存区，不能越界偷窥其他进程的数据——通过基址/限长寄存器或页面保护位实现。

### 程序的装入和链接

程序从源代码变成可在内存中运行的进程，经历两个阶段：

**装入方式** 🔥：
- **绝对装入** 💀：编译时就确定所有内存地址——程序之后只能在预先分配的那些物理地址上运行。早期单道 OS 使用。
- **可重定位装入（静态重定位）** 🔥：程序编译偏移从 0 开始，装入时根据分配的起始物理地址一次性修正所有地址引用——装入后不能再移动。
- **动态运行时装入** 🔥 🔥：程序开始执行后地址转换持续在每次访存时由硬件重定位寄存器完成——程序运行过程中可在内存中搬移（交换）。现代 OS 普遍采用这种方式。

**链接方式** 🔥：
- **静态链接**：所有目标模块+库被编译程序统一链接成一个完整可执行文件——文件大且共享共用库被重复占用。传统 .EXE 编译。
- **装入时动态链接**：先将各模块调到物理内存里再记录符号链接关系——类似共享库的装入时确定。
- **运行时动态链接** 🔥：程序运行到要调用某库时才从磁盘找该库加载到内存并链接。现代 OS 全部负载于此方式——典型的 Windows DLL 和 Linux .so 加载机制。

### 逻辑地址与物理地址

**逻辑地址（虚地址）** 🔥：CPU 产生的地址——每道程序在以 0 为起点的逻辑地址空间中编写。**物理地址**：主存单元的硬地址——逻辑地址在访存前必须转换为物理地址。这个转换过程是计算机组成原理 MMU 与操作系统页表的协作结果，也是后续所有内存管理的讨论中心。

### 内存保护

最简单的保护就是给每个进程一对**基址/限长寄存器**——基址寄存器存该进程分配内存区的启始物理地址，限长寄存器存该内存区的长度。每次进程访问内存，硬件自动检查：`基址 ≤ 访问地址 < 基址 + 限长`，犯界则产生地址越界中断。

> 更细粒度的保护在页式/段式系统中实现——每页/段有保护位（读/写/执行）限制该进程对该页的访问方式。

## 连续分配管理方式

系统将物理内存划分成若干连续的分区，进程占据一整块分区。

### 单一连续分配

单一连续分配 💀 是最原始简单的分配方式——内存只分两道：低地址部分全给 OS，剩余全给一个用户进程使用。每次同时只能有一道用户作业。早期 MS-DOS 就是这种方式。已完全过时。

### 固定分区分配

固定分区 🔥 把用户内存预先划分为若干固定大小的区域——每个区可以装一道作业。分区分大小相等（用完不能再接多作业的浪费）或不等（提供不同大小选择减低浪费）。分区个数和大小不可动态更改——灵活性差。优点是实现极简单。可用**分区使用表**管理各分区占用情况。

> 💀 已在现代系统中几乎消失（嵌入式有使用），408 了解分区等大小的浪费为主。

### 动态分区分配

动态分区 🔥 根据进程实际需要大小实时从空闲内存区划出一块——分区大小、数量完全动态。当进程结束时释放回回收插入空闲区表——此时可能出现大量不连续的小碎片（外碎片）。分配算法：

- **首次适应（First Fit）** 🔥：沿空闲分区链从头找，找到第一个 ≥ 需求的空闲区就分。速度快，查找开销小。但低地址区易累积大量小碎片区——下次分配搜索要经过一串小程序拒绝才找到目标。
- **最佳适应（Best Fit）**：找所有空闲区中大小最适合（刚好不小于需求尺寸且最小的那个）。分配后剩余空闲区最小——看似最佳但其实生成大量极小的无用碎片越来越碎片化查找变慢。
- **最坏适应（Worst Fit）**：每次分配最大的空闲区——剩余空闲区依然较大。看似能避免微碎片但反而导致大空闲区过快缩小而使后续分配大空间请求失败。
- **邻近适应（Next Fit）**：和首次适应一样，但从上一处分配位置的继续往下找——平衡了低地址和高地址碎片的分布。

> 408 经常考试：给当前的空闲区链和分配或释放序列，问每种算法在哪处分配、事后空闲区链的状态变化。

### 可重定位分区分配

可重定位分区 🔥 通过硬件能力动态**紧凑**——将内存中所有运行中的进程移动使其分区聚拢到一端，另一端的空闲区合并成一个大区块供重新使用。但搬动程序开销极大，实际中极少做。多数 OS 用「页式」回避外碎片问题。

### 动态分区分配的回收

释放一个分区后检查它两邻区是否也空闲——若相邻空闲就可合并成一个更大的空闲区以减少碎片。考研里的算法就是边界标记——在每个分区的头和尾保存分区大小和是否空闲的标记以便合并时定位相邻分区。

## 非连续分配管理方式

### 基本分页存储管理

分页存储 🔥 🔥 是现代操作系统内存管理的基石。将物理内存切成大小相等的固定块——**页框（Frame）**，逻辑地址的进程也切成同样大小的**页面（Page）**。进程的页面可以非连续地装入任何一个空闲的页框中，由每个进程的一张**页表**记录映射关系：页号 → 框号。

**地址变换机构** 🔥 🔥：CPU 的虚地址（设为 32 位）分为**页号 p**（高位）和**页内偏移 d**（低位）。MMU 的页表基址寄存器存有页表首地址 → 查到页号 p 对应的框号 f → 物理地址 = f × 页面大小 + d。这个过程可由快表大幅降低因查表需要的一次额外访存。

**快表（TLB）** 🔥 在计算机组成原理已详述——这里再次强调它的位置：TLB 在地址转换的第一步就被查询——命中直接拿到框号不需访问页表；不命中才访问页表并将本次转换的页号-框号对装入 TLB。

**多级页表** 🔥：64 位系统地址空间超出单级页表的表示——采用多级分层页表。每个级别的页表本身也是放在一个页框里的数据——按需建页表，内存只维护真正在使用的那部分页表，省下大量页表预留空间。

**页表项大小与页内偏移的计算** 🔥：408 大题的核心送分计算——给定页面大小（如 4KB = 2^12）、地址长度（32 位），偏移 d 固定 = 12 位（= 页面大小）。页号 p = 32 − 12 = 20 位——所以最多 2^20 页面框，页表最多 1M 项。如果是 64 位系统需要更多层次分解。

### 基本分段存储管理

分段管理 🔥 和页式完全不同——不是等分内存而是按程序的逻辑段分开分配给自然边界（如代码段、数据段、栈段、堆段）。每个段长度不等，有一张段表记录各段的基地址和界限。地址转换涉及段号和段内偏移——段表中需查界限确保访问不越界。

分段对比分页的差异内存：
| 对比 | 分页 | 分段 |
|------|------|------|
| 单位 | 等长页（物理固定） | 不等长段（逻辑意合单位） |
| 地址结构 | 一维——页号+偏移 | 二维——(段号, 段内偏移) |
| 可见性 | 用户不可见（硬件施作） | 用户可见（编译器设段） |
| 碎片 | 内碎片（每程序恰好分页） | 外碎片（段参差破碎） |
| 保护 | 按页保护（粗粒度） | 按段类型保护（自然） |

> 💀 纯粹的分段现在基本不用——x86 的分段已被软化到退化作用，现代系统以页式或多级页表为主。

### 段页式存储管理

段页式 🔥 是分段与分页的结合——先把程序按逻辑分成若干段，每段内部再进行分页。地址变成三维结构：(段号, 段内页号, 页内偏移)。进程获得一个段表指向各段的页表地址，页表再指物理页。段页式的优点有所有：从段层面保护、从页层面无外碎片。缺点：地址转换需要两次查表（段表 + 页表），比纯页式又慢了一步——所以 TLB 也更重要得多。

> 408 对段页式考得很少，停留在画地址结构图和推两级翻译步骤即可。

## 虚拟内存管理

### 虚拟内存的基本概念

虚拟内存 🔥 🔥 是一种内存管理技术，让进程产生一个幻觉——它拥有连续可用的完整大片内存，但实际物理内存可能只有其中一部分。其理论基础正是**程序的局部性原理**——程序在某一时间片段中仅频繁使用一小部分页面（时间局部性）和相邻页面被连续需要（空间局部性）。虚拟内存能让物理主存中仅驻留进程的活跃页面，不活跃页面暂存在外存交换区，用到时再调入。

虚拟内存的三个基本特征（与 OS 四大特征呼应）：
1. **多次性**：程序无需一次性装入完整内存，可以分多次调入必需的页面（分散加载）。
2. **对换性**：运行过程中可把暂时不需要的页换出到磁盘再将急需页换入。
3. **虚拟性**：逻辑上对用户的效应是地址空间远大于物理内存——虚拟化存储。

### 请求分页管理方式

请求分页 🔥 🔥 是虚拟内存的最基础实现——在基本分页的基础上加载页面调度能力。页表除了页框号外，还要扩展若干个控制位：

- **状态位（有效位）** 🔥：指示该页面是否已在物理内存中。1 已加载，0 不在（后续访问引发缺页中断）。
- **访问字段**：记录该页的最近被访问情况（用于置换算法选择牺牲页）。
- **修改位（脏位）** 🔥：如果某物理页自从调进内存后被写过，置 1 ——写盘时已被修改就需回写外存，没有修改的直接覆盖掉。
- **外存地址**：纪录该页面在外存交换区的地址——缺页处理时从该地址把页面调入。

**缺页中断机构** 🔥 🔥：当 CPU 访问一页但发现其状态位为 0（未在内存），MMU 会立即触发缺页中断陷入核心。缺页中断处理流程：① 检查该页的可访问权限 ② 寻找一个空闲页框（无空闲时通过置换算法牺牲一个）③ 发出磁盘 I/O 把该页读到页框 ④ 恢复引起的指令执行——注意：是重新执行整条引发缺页的指令而不是继续下一指令（缺页是故障型，不同于普通中断）。

### 页面置换算法

当物理页框全被占用又需要调入新页时，就必须淘汰一个已占用页框——**页面置换算法**决定淘汰谁。408 的期望是对每种算法算出给定页面引用序列的缺页次数计算 🔥。

**最佳置换（OPT）** 💀：淘汰将来**最晚用到的那一页**。从未来看最完美——可惜无法预测未来。用于事后比较——其他算法和 OPT 距离越小越好。

**先进先出（FIFO）**：淘汰最早调入的页面。用 FIFO 队列实现简单，缺点是性能常很差——可能有 **Belady 异常** 🔥（增加页框数反而缺页次数增加）。408 每年必出 Belady 异常的选择判断题——FIFO 是唯一出现 Belady 异常的算法。

**最近最久未使用（LRU）** 🔥 🔥：淘汰目前最久没被访问的页面。性能接近 OPT（堆栈类算法完美免 Belady 异常），但纯硬件实现 LRU 需要跟踪每个页框最近的访问时间——代价高。在实践中通过近似的 CLOCK 算法实现。

**时钟置换（CLOCK / NRU）** 🔥 🔥：每个物理页框配一位访问位（使用位）。循环扫描物理页框，遇到访问位为 1 就清 0 继续转，遇到访问位 0 的访问位都没有被最近用过——淘汰它。CLOCK 是 LRU 的极低代价近似，是现代 OS 实际使用的页面置换算法 🔥。

**改进型 CLOCK**：在 CLOCK 的基础上加入修改位——优先淘汰既没被访问也没被修改(0,0)的页面（纯粹丢弃不写盘），第二次转才考虑(0,1)或(1,0)、(1,1)组合。降低了写盘的次数。

> 408 页面置换大题考点统合：给一串页面引用和分配页框数，用 FIFO/LRU/CLOCK 每一次算缺页——画置换过程格子。

### 页面分配策略

决定每个进程应获多少个物理页框，分别为以下三种策略 🔥：
- **固定分配 + 局部置换**：每进程分配固定数量页框——缺页时只能淘汰自己占的某页。实现最简，但低负载浪费、高负载猛烈缺页。
- **可变分配 + 全局置换**：OS 维护一个全局空闲页池——任何进程缺页时都可从池中获取页框并分配该新页。页框可在进程间自由流动——但需要复杂的监控机制防抢占过度。
- **可变分配 + 局部置换** 🔥：每进程缺页率高时额外分配新页框以减轻缺页；若缺页率低就回收空闲页——按需调节。实际现代 OS 的最佳候选（如 Linux 的页面回收与工作集平衡机制）。

### 抖动（颠簸）和工作集

**抖动（颠簸 — Thrashing）** 🔥 🔥：当系统的物理内存总量小到不足以支持同时运行进程的活跃页面工作集时，系统频繁陷入接二连三的缺页中断——CPU 的大量时间全都耗在页面换进换出上（CPU 利用率暴降到近零）。解决抖动的核心是减少并发进程数量或增加物理内存。

**工作集模型** 🔥：即一段时间内（时间窗口 Δ）进程频繁访问的页面的集合。通过测量和维持每个进程所需的最小工作集页框数，系统可控制在何种多进程并行度下内存和 CPU 利用率最优——一旦总可分配页框 < 各进程工作集之总和——立刻触发拥塞控制并拒绝新进程进入以减少并发度。工作集是防治抖动的最佳理论模型。

> 408 会有一道选择题问你抖动的原因和解决方案——「物理内存不足导致频繁缺页」是标准答案。

## 常见题型

- **页式存储的地址变换计算（大题重中之重）**：给虚地址 (十进制/十六进制)，以及页表/帧表的数据，解答需拆出页号和偏移，查页表求出物理地址。熟记页面大小 = 2^偏移位数。
- **页面置换算法的缺页率计算**：给访问序列和给定的物理页框数，FIFO/LRU/CLOCK 各自手动计算每一步缺页情况——缺页次数除以总步长 = 缺页率。
- **各种动态分区分配算法的模拟**：给空闲分区链和分配/释放序列，画出各算法下的空闲区状态的变换——FF/BF/WF/NF。
- **段页式地址变换**：（偶尔考）给出三段号加页号加偏移，一级段表中查某段的页表地址，第二步页表中查到物理框号合并偏移。
